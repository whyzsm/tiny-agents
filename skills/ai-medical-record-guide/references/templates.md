# 病案首页与出院小结模板

> 本文档定义住院病案首页和出院小结的标准化 Markdown 输出模板。
> 依据《卫生部关于修订住院病案首页的通知》（卫医政发〔2011〕84号）及实际电子病历规范制定。
> **此文档可被替换以适配不同医疗机构的格式要求。**

### 病案首页填写规则

1. **主要诊断选取**：对患者本次住院治疗影响最大、花费医疗资源最多的诊断作为主要诊断
2. **入院病情判定**：
   - "有"：入院时即明确的诊断
   - "临床未确定"：入院时疑似但未确认
   - "情况不明"：入院时未考虑到
   - "无"：住院期间新发生的情况
3. **住院天数计算**：出院日期 - 入院日期（入院日和出院日只算1天）
4. **ICD编码**：如模型无法确定精确编码，标注 `[待编码]` 提示人工补充


### 出院小结填写规则

1. **诊断一致性**：出院诊断应与病程记录中最后确认的诊断一致
2. **治疗经过精炼化**：不需逐日抄录病程，提取关键诊疗节点
3. **出院用药完整性**：与出院处方逐项核对
4. **红线症状明确化**：基于疾病特征给出具体、可操作的警示症状
5. **表达独立性**：出院小结的文本表述应独立于入院记录和病程记录，不直接复制，而是基于同一事实重新组织语言

## 三、模板变量说明

- `{{变量名}}`：需从病历数据中填充的占位符
- 若某字段信息缺失且无法推断，填入 `[信息缺失]` 并在校验报告中标记
- 若某区段完全不适用（如无手术患者的"手术及操作信息"），保留表头并填"无"
- 费用汇总为可选区段，无费用数据时整段省略
