# 病历校验规则引擎

> 本文档定义 AI 智能病历系统的全量校验规则。
> 校验分为三个层级：P0（阻断级/安全红线）、P1（警告级/质量问题）、P2（提示级/完善建议）。
> **此文档可被替换或扩展以适配不同医疗机构的质控标准。**

---

## 一、校验层级定义

| 层级 | 含义 | 处理方式 |
|------|------|---------|
| 🔴 P0 | 安全红线，可能危及患者安全 | 必须醒目标注，阻断生成直到确认 |
| 🟡 P1 | 质量问题，影响病历合规性 | 警告提示，允许用户确认后继续 |
| 🔵 P2 | 完善建议，提升病历质量 | 温和提示，不阻断流程 |

---

## 二、P0 安全校验规则

### 2.1 过敏-用药冲突检测

- **规则**：处方中的药品/药物成分不得与过敏史中记录的过敏原匹配
- **匹配逻辑**：
  - 精确匹配：过敏原名称 ⊂ 药品名称（如过敏"青霉素" → 处方含"青霉素V钾"）
  - 类别匹配：同类药物交叉过敏（如过敏"青霉素" → 处方含"头孢"类需警告）
  - 成分匹配：过敏原 ⊂ 药物成分表（如过敏"磺胺" → 处方含"复方新诺明"）
- **输出格式**：
  ```
  🔴 P0-ALLERGY：检测到过敏-用药冲突
  - 过敏原：{{过敏原}}
  - 冲突药品：{{药品名称}}
  - 冲突类型：精确匹配/类别交叉/成分匹配
  - 建议：立即核实，必要时更换药物
  ```

### 2.2 皮试合规性校验

- **规则**：需皮试药物必须有皮试结果记录
  - 皮试标识为 true 且缺少皮试结果 → P0
  - 皮试结果为"阳性"但仍开具该药 → P0
  - 皮试时间距给药超过24小时 → P0
- **输出格式**：
  ```
  🔴 P0-SKINTEST：皮试校验未通过
  - 药品：{{药品名称}}
  - 问题：{{具体问题}}
  - 建议：{{处理建议}}
  ```

### 2.3 关键必填字段缺失

- **规则**：以下字段在对应场景中缺失视为 P0
  - 过敏史（任何场景）
  - 入院诊断（住院场景）
  - 出院诊断（出院场景）
  - 主诉（任何场景）
- **输出格式**：
  ```
  🔴 P0-REQUIRED：关键安全字段缺失
  - 缺失字段：{{字段名}}
  - 所属模块：{{模块名}}
  - 风险说明：{{说明}}
  ```

## 三、P1 质量校验规则

### 3.1 诊断一致性校验

- **入院-出院诊断对比**：
  - 出院诊断应包含入院诊断中的主要诊断（或明确标注变更原因）
  - 新增出院诊断应在病程记录中有依据
  - 入院诊断被删除的应在病程记录中说明
- **处方-诊断匹配**：
  - 处方中的药品应与诊断相关（至少不矛盾）
  - 超说明书用药应有特殊标注

### 3.2 时间逻辑校验

- 入院时间 < 首次病程时间 ≤ 后续病程时间（按顺序递增）
- 入院时间 < 出院时间
- 手术时间在入院和出院之间
- 医嘱时间在入院和出院之间
- 住院天数 = 出院日期 - 入院日期

### 3.3 必填字段完整性

- 各模块中标记为"✅"的字段缺失 → P1
- 条件必填字段在满足条件时缺失 → P1
- **输出格式**：
  ```
  🟡 P1-MISSING：必填字段缺失
  - 模块：{{模块名}}
  - 缺失字段：{{字段列表}}
  - 说明：{{缺失影响}}
  ```

### 3.4 生命体征范围校验

- 体温：35.0-42.0℃，超出标记异常
- 脉搏：<60 或 >100 标记异常
- 呼吸：<12 或 >20 标记异常
- 血压：收缩压 <90 或 >180、舒张压 <60 或 >120 标记异常
- **异常不一定错误**，但需确认是否与临床情况一致

### 3.5 药品剂量合理性校验

- 单次剂量 × 频次 应与总量一致
- 剂量超出常规范围需警告（依赖知识库，无法确认时提示人工审核）
- 静脉用药应有滴速

### 3.6 风险评估完整性

- 住院患者缺少压疮/跌倒/VTE/营养评估 → P1
- 评分结果为高风险但无对应预防医嘱 → P1

## 四、P2 完善建议规则

### 4.1 推荐字段缺失

- 标记为"⚠️ 推荐"的字段缺失 → P2
- 联系方式缺失（影响随访）→ P2

### 4.2 文本质量建议

- 主诉格式不规范（缺少"症状+时间"结构）→ P2
- 既往史/家族史写"无"而非标准否认用语 → P2
- 病程记录过于简短（<50 字）→ P2

### 4.3 随访计划完整性

- 缺少复诊时间 → P2
- 缺少红线症状 → P2
- 缺少居家监测指标（慢病患者）→ P2

### 4.4 表达独立性检查

- 住院不同阶段的病历文本不应直接复制
- 出院小结中的表述应独立于入院记录
- 同一事实在不同文档中应转换表达方式

## 五、校验报告输出格式

校验完成后，按以下格式输出校验报告：

```markdown
## 📋 病历质量校验报告

**校验时间**：{{时间}}
**病历类型**：{{门诊/住院-具体阶段}}
**患者**：{{姓名}} | {{住院号/门诊号}}

### 校验结果概览

| 层级 | 数量 | 状态 |
|------|------|------|
| 🔴 P0 安全红线 | {{数量}} | {{全部通过 / 存在问题}} |
| 🟡 P1 质量问题 | {{数量}} | {{全部通过 / 存在问题}} |
| 🔵 P2 完善建议 | {{数量}} | — |

### 详细问题列表

#### 🔴 P0 安全红线（如有）
{{逐条列出}}

#### 🟡 P1 质量问题（如有）
{{逐条列出}}

#### 🔵 P2 完善建议（如有）
{{逐条列出}}

### 校验结论
{{总结性结论：是否可通过 / 需要修正的关键项}}
```

## 六、校验规则扩展指南

如需添加自定义校验规则：

1. 确定校验层级（P0/P1/P2）
2. 定义规则名称和编码（如 `P1-CUSTOM-001`）
3. 描述触发条件
4. 定义输出格式
5. 将规则追加到本文档对应层级章节中
