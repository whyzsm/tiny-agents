# 灰色地带判定指南

## 什么是灰色地带

灰色地带指：法律未明确禁止，但存在合规风险、道德争议或监管不确定性的商业操作。

**核心原则：灰色地带 ≠ 绿灯。默认暂停，需用户明确确认后方可继续。**

## 灰色地带分类

### A类：高风险灰色（强烈建议不碰）

| 类型 | 示例 | 风险说明 |
|------|------|---------|
| 收益模式不透明 | 需要隐瞒真实收入来源的项目 | 可能涉及洗钱或非法集资 |
| 虚假身份操作 | 要求使用虚假身份注册/交易 | 涉嫌欺诈 |
| 隐藏交易路径 | 资金流向刻意复杂化 | 反洗钱法风险 |
| 未注册证券代币 | 发行未备案的代币/积分 | 可能被认定为非法证券 |
| 自动化批量操作 | 批量注册账号、自动发帖/评论 | 违反平台规则，可能触犯计算机犯罪法 |

### B类：中风险灰色（需用户确认后谨慎执行）

| 类型 | 示例 | 风险说明 |
|------|------|---------|
| 用户不知情的自动化 | 后台自动收集公开数据 | 边界模糊，需评估robots.txt和数据性质 |
| 新型商业模式 | 法律尚无明确规定的创新业务 | 监管可能随时收紧 |
| 跨境业务合规 | 面向海外用户但运营在中国 | 需同时考虑两国法律 |
| 知识产权边界 | AI生成内容的版权归属 | 法律仍在演进中 |
| 竞价/套利行为 | 利用信息差在平台间套利 | 需确认不涉及操纵市场 |

### C类：低风险灰色（提醒后可执行）

| 类型 | 示例 | 风险说明 |
|------|------|---------|
| 税务申报提醒 | 收入达到起征点需申报 | 提醒用户及时申报即可 |
| 平台条款变化 | 平台更新规则导致原有操作违规 | 定期检查平台规则 |
| 内容审核边界 | 内容可能被误判为违规 | 建议预留申诉通道 |

## 判定流程

```
收到商业策略
    │
    ▼
第一层黑名单筛查 ──命中──→ ❌ 绝对禁止，终止
    │
    未命中
    │
    ▼
对照A/B/C分类表
    │
    ├── 命中A类 → ⚠️ 高风险预警，强烈建议不执行
    │              输出报告，等待用户明确指令
    │
    ├── 命中B类 → ⚠️ 中风险预警，需用户确认
    │              输出报告，说明风险，等待用户决策
    │
    ├── 命中C类 → 💡 低风险提醒，可执行但需注意
    │              输出提醒，附注意事项
    │
    └── 未命中任何灰色分类 → ✅ 通过
```

## 熔断机制

- **单日2次灰色预警** → 自动进入安全模式，暂停所有新策略探索
- **单日1次A类预警** → 立即暂停相关策略，通知用户
- **累计5次灰色预警（7天内）** → 建议全面合规复查

## 预警记录格式

```markdown
[合规预警]

触发操作：{具体操作描述}
风险类型：{A类高风险 / B类中风险 / C类低风险}
涉及条款：{相关法律条文或分类编号}
建议：{暂停 / 需要用户确认 / 需要补充资质}
替代方案：{合法的替代路径，如有}
```

## 常见灰色场景判定参考

### "帮人代写文章赚钱"
→ B类：如果是一般性写作辅助，低风险；如果是代写学术论文，涉及学术诚信问题，B类中风险

### "用AI批量生成内容发到平台赚流量费"
→ A类：批量自动化操作+可能涉及虚假内容，高风险

### "做信息差套利（如跨境电商选品信息）"
→ C类：信息整理本身合法，但需注意数据来源合规

### "开发工具帮别人做合规检查"
→ C类：工具本身合法，但需声明不构成法律意见

### "在多个平台注册账号做分销"
→ B类：如涉及批量注册，A类；如正常注册做分销，C类；需具体分析
