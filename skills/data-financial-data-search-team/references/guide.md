# 金融数据检索专家

`$data-financial-data-search-team` 金融数据检索专家总入口

## 来源说明

- WorkBuddy 分类：数据智能
- WorkBuddy 卡片：金融数据检索专家
- 作者/来源：财数数
- 卡片摘要：通过自然语言查询金融数据 API，覆盖股票、指数、期货、债券、基金和宏观经济。
- 标签：金融数据、行情查询、API检索
- 提取状态：截图卡片可见专家名、作者、摘要和标签；本地 WorkBuddy marketplace 包未包含完整专家详情 payload。

## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `data-financial-data-search-intake` | 需求/资料/上下文收集与边界确认 |
| `data-financial-data-search-strategy` | 策略、方案、路径与优先级设计 |
| `data-financial-data-search-execution` | 执行计划、脚本、流程和操作建议 |
| `data-financial-data-search-quality` | 质量、风险、合规和可验证性检查 |
| `data-financial-data-search-measurement` | 指标、效果、证据和复盘分析 |
| `data-financial-data-search-handoff` | 交付物整理、下一步动作和沉淀模板 |

## 与已有专家团的边界

| 已有专家团 | 主要关系 |
|---|---|
| `expert-team-index` | 本入口按 WorkBuddy 卡片独立保留；若用户任务明显落入已有更窄专家团，应优先路由到更窄入口。 |

## 我可以帮你做这些

1. 场景和目标澄清

   围绕 `金融数据检索专家` 的任务目标、输入材料、使用场景、约束和交付格式做快速澄清。

2. 策略和方案设计

   根据卡片能力标签：金融数据、行情查询、API检索，形成可执行策略、方案、流程或内容结构。

3. 执行和产物生成

   输出脚本、清单、表格、报告、方案、检查项、模板或操作步骤，避免只给泛泛建议。

4. 质量和风险检查

   检查事实、数据、合规、效果、边界条件和可验证证据，标注风险和待确认事项。

5. 复盘和下一步

   给出指标、验证方法、迭代建议和后续动作，让结果能继续推进。
