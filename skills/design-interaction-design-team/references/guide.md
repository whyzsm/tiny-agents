# 交互设计专家团

`$design-interaction-design-team` 交互设计专家团总入口

## 触发的技能

| 技能 | 能力 |
|---|---|
| `$design-interaction-design-team` | 交互设计专家团总入口 |
| `$ux-researcher` | 用户画像、旅程地图与研究洞察 |
| `$ui-ux-pro-max` | 信息架构、视觉风格与设计系统 |
| `$ui-audit` | UX 可用性审计与优化建议 |
| `$ui-ux-design` | 微交互、动效与响应式设计规范 |
| `$frontend-design` | 高质量前端界面实现 |
| `$oliver-accessibility` | WCAG 无障碍合规实现 |

## 我可以帮你做这些

你现在要完成一项全面的交互设计任务。你已安装以下 Skill，请按步骤串联使用：

## 步骤 1：用户研究与画像构建（获取层）
使用 **UX Researcher** 完成：
- 基于数据驱动方法生成用户画像（Persona）
- 绘制用户旅程地图（Journey Map），标注关键触点和情绪曲线
- 设计可用性测试框架和测试任务
- 识别用户痛点、需求和行为模式
- 产出用户研究洞察报告

输出用户画像文档、旅程地图和研究洞察。

## 步骤 2：信息架构与交互方案设计（分析层）
使用 **UI/UX Pro Max** 完成：
- 设计信息架构（IA），定义内容层级和导航结构
- 制定视觉风格方向（色彩/字体/间距/图标风格）
- 设计组件规格和交互状态（默认/悬停/按下/禁用/加载）
- 编写界面文案和微文案（按钮/提示/错误信息）
- 构建设计系统令牌和组件文档

输出信息架构图、交互规格和设计系统文档。

## 步骤 3：UX 可用性审计（分析层）
使用 **UI Audit** 完成：
- 基于 UX 成熟原则自动审计界面设计
- 评估视觉层级是否清晰（主次关系、视觉引导）
- 检查认知负荷是否合理（信息密度、决策复杂度）
- 审查导航设计（可发现性、一致性、回退路径）
- 评估无障碍性基础指标

输出 UX 审计报告和优化建议清单。

## 步骤 4：微交互与动效设计（分析层）
使用 **UI/UX Design Guide** 完成：
- 设计移动优先的交互模式和手势方案
- 定义微交互（按钮反馈、加载动画、状态切换）
- 制定色彩系统和字体排版层级
- 确保设计符合 WCAG 2.2 无障碍标准
- 集成 Tailwind + Shadcn/ui 的交互组件规范

输出微交互规范和动效设计方案。

## 步骤 5：交互原型优化与实现（输出层）
使用 **Frontend Design** 完成：
- 结合前序审计结果优化美学、动效和用户体验
- 确保交互动效流畅自然（缓动曲线、时序控制）
- 优化界面视觉细节（阴影、渐变、过渡）
- 生成视觉美观、体验良好的前端界面代码
- 根据目标风格和可用性反馈迭代界面细节

输出优化后的前端交互代码和实现说明。

## 步骤 6：无障碍合规实现（输出层）
使用 **Accessibility** 完成：
- 实现语义化 HTML 结构和适当的 ARIA 标签
- 确保键盘导航完整（Tab 顺序、焦点管理、快捷键）
- 验证颜色对比度（文本 4.5:1，大文本 3:1）
- 测试屏幕阅读器兼容性
- 实现实时区域（Live Region）和表单标签

输出 WCAG 2.1 AA 合规的前端代码。

## 最终输出
将以上步骤的结果整合为完整的交互设计包，交付以下文件：
1. **用户研究报告**：用户画像、旅程地图、痛点洞察
2. **信息架构与交互规格**：IA 图、组件规格、状态定义
3. **UX 审计报告**：视觉层级/认知负荷/导航评估和优化建议
4. **微交互与动效方案**：交互模式、动效规范、时序定义
5. **前端交互代码**：优化后的高质量界面实现
6. **无障碍合规报告**：WCAG 2.1 AA 检查结果和合规代码
