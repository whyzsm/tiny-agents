# PRD 写作专家团

`$design-prd-writing` 端到端产品需求文档写作专家团入口

## 触发的技能

| 技能 | 能力 |
|---|---|
| `$design-prd-writing` | 完整 PRD 写作工作流总入口 |

## 我可以帮你做这些

1. 需求结构化

   调用 `design-prd-writing` 的需求分析阶段

   把初始想法整理为用户、场景、需求、依赖、用户故事和验收标准。

2. 优先级和 MVP 定义

   调用 `design-prd-writing` 的产品规划阶段

   使用 RICE、MoSCoW、Kano 等方法拆分优先级，明确 MVP 和延后范围。

3. 可实施 PRD 拆解

   调用 `design-prd-writing` 的 PRD 拆解阶段

   生成面向研发落地的 PRD 条目、任务清单、用户故事、验收标准和依赖顺序。

4. 专业 PRD 成稿

   调用 `design-prd-writing` 的 PRD 写作阶段

   输出包含背景、目标、成功指标、功能需求、非功能需求、边界和发布考虑的 PRD。

5. 设计交付转换

   调用 `design-prd-writing` 的设计交付阶段

   将 PRD 转换为设计目标、信息架构、交互流程、页面要求、状态要求和交付清单。

6. PRD 评审优化

   调用 `design-prd-writing` 的评审阶段

   对 PRD 进行评分，标出缺失证据、风险、扣分项和可执行修改建议。

## 完整交付物通常是

```text
结构化需求清单

用户故事和验收标准

优先级评分表和 MVP 范围

专业 PRD 文档

设计交付文档

PRD 评审报告和修改建议
```

## 你可以这样用我

```text
$design-prd-writing 帮我把这个产品想法整理成完整 PRD，并标出 MVP 范围

$design-prd-writing 将这些零散需求拆成用户故事、验收标准和研发任务顺序

$design-prd-writing 把这份 PRD 转成设计交付文档，包含信息架构和页面要求

$design-prd-writing 审查这份 PRD，给出评分、扣分依据和改进建议
```
