# 产品战略专家团

Use this skill as the routing entry point for product strategy and product management work. It packages the WorkBuddy product-management team workflow into a Codex skill entry instead of requiring the WorkBuddy plugin to be installed.

## Workflow

1. Read `references/guide.md` to classify the request, source inputs, and expected artifacts.
2. Use the narrowest relevant WorkBuddy capability module from the guide. For full-package requests, follow `references/workflow.md` in order.
3. Ask only for missing product context that blocks the next useful output: target users, problem evidence, business goals, constraints, timeline, or available metrics.
4. Keep assumptions explicit. Prefer tight MVP scope, measurable success metrics, explicit non-goals, and evidence-backed recommendations.
5. Return the concrete artifacts requested by the user, plus open questions, risks, confidence level, and next-step suggestions.

## Source Capability Modules

- `product-management-workflows`
- `feature-spec`
- `roadmap-management`
- `stakeholder-comms`
- `user-research-synthesis`
- `competitive-analysis`
- `metrics-tracking`
- `product-brainstorming`
- `sprint-planning`

## Output

Produce only the deliverables relevant to the request. For a full product strategy package, assemble the output set described in `references/workflow.md`.
