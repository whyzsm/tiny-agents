# 产品战略专家团

`$design-product-strategy-team` 产品战略专家团总入口

## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `product-management-workflows` | 串联功能规格、路线图、利益相关者沟通、用户研究、竞品分析和指标审查 |
| `feature-spec` | 输出问题陈述、目标/非目标、用户故事、需求分级、验收标准和成功指标 |
| `roadmap-management` | 使用 RICE、MoSCoW、ICE、价值/工作量矩阵等方法规划路线图 |
| `stakeholder-comms` | 面向高管、工程、跨职能伙伴、客户或董事会撰写状态更新和决策沟通 |
| `user-research-synthesis` | 将访谈、问卷、工单、行为数据整理为主题、画像、机会和建议 |
| `competitive-analysis` | 生成竞品概览、功能对比矩阵、定位分析、SWOT 和战略启示 |
| `metrics-tracking` | 定义北极星指标、L1/L2 指标、OKR、仪表盘和指标审查节奏 |
| `product-brainstorming` | 探索问题空间、发散方案、挑战假设并收敛产品方向 |
| `sprint-planning` | 估算容量、拆分 P0/P1/P2、识别依赖和风险，生成冲刺计划 |

## 与已有专家团的边界

| 已有专家团 | 主要覆盖 | 与本团的区别 |
|---|---|---|
| `design-prd-writing` | 从需求发现到 PRD 成稿、设计交付和 PRD 评审 | 本团更偏产品管理闭环，额外覆盖路线图、利益相关者沟通、竞品、指标和冲刺计划 |
| `design-interaction-design` | 用户研究、信息架构、交互规范、UX 审计、无障碍和前端实现 | 本团不负责 UI/交互细节，优先回答产品方向、范围、优先级和决策沟通 |
| `design-ui-prototype` | 从需求或设计稿到设计系统、线框图和高保真 HTML 原型 | 本团的输出停在产品策略和管理交付物，原型落地交给 UI/设计类专家团 |

## 我可以帮你做这些

1. 产品机会和问题空间探索

   使用 `product-brainstorming`

   从模糊想法、用户痛点、业务目标或市场机会出发，识别目标用户、根因、约束、假设和可验证方向。

2. PRD 与功能规格

   使用 `feature-spec`

   将产品机会转成结构化 PRD：问题陈述、目标、非目标、用户故事、需求分级、验收标准、成功指标和开放问题。

3. 产品路线图规划

   使用 `roadmap-management`

   根据目标、约束、团队容量和依赖，选择 Now/Next/Later、季度主题或 OKR 对齐路线图，并给出优先级依据。

4. 用户研究综合

   使用 `user-research-synthesis`

   将访谈、问卷、工单、可用性测试、NPS 或行为数据整理为主题、证据、用户分群、机会领域和建议。

5. 竞品和定位分析

   使用 `competitive-analysis`

   对竞争对手做功能矩阵、定位拆解、优势劣势、机会威胁和战略影响分析。

6. 指标审查和决策仪表盘

   使用 `metrics-tracking`

   定义北极星指标、健康指标、诊断指标、OKR 和审查节奏，并给出趋势判断和行动建议。

7. 利益相关者沟通

   使用 `stakeholder-comms`

   为高管、工程、客户、董事会或跨职能伙伴生成进展更新、风险沟通、发布说明和决策记录。

8. 冲刺规划

   使用 `sprint-planning`

   基于团队容量、backlog、carryover、依赖和目标，生成冲刺计划、风险表和 Definition of Done。

## 完整交付物通常是

```text
产品机会/问题空间分析
PRD 或功能规格
优先级评分表和 MVP 范围
产品路线图
用户研究综合报告
竞品分析简报
产品指标审查和行动建议
利益相关者沟通稿
冲刺计划
```
