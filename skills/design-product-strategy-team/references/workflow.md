# 产品战略工作流

你现在要完成一项产品管理和产品战略任务。本包从 WorkBuddy product-management 团队能力转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：问题空间探索（获取层）

使用 **product-brainstorming** 完成：
- 明确目标用户、使用场景、痛点和替代方案
- 区分症状和根因，列出关键假设
- 用 How Might We、JTBD、机会解决方案树等方法打开方案空间
- 给出最小验证实验和需要补充的证据

输出产品机会分析、关键假设和验证计划。

## 步骤 2：PRD 与功能规格（分析层）

使用 **feature-spec** 完成：
- 写清问题陈述、目标和非目标
- 按角色整理用户故事和边界条件
- 将需求分为 P0/P1/P2 或 MoSCoW 等优先级
- 输出验收标准、成功指标、开放问题和时间线考虑

输出可评审 PRD 或功能规格文档。

## 步骤 3：路线图和优先级规划（分析层）

使用 **roadmap-management** 完成：
- 选择 Now/Next/Later、季度主题、OKR 对齐或时间线视图
- 使用 RICE、MoSCoW、ICE 或价值/工作量矩阵评估优先级
- 明确依赖、风险、资源容量和下游影响
- 标出当前承诺、下一阶段计划和延后范围

输出路线图、优先级表和取舍说明。

## 步骤 4：研究和竞品证据补强（分析层）

使用 **user-research-synthesis** 和 **competitive-analysis** 完成：
- 汇总访谈、问卷、工单、NPS、可用性测试或行为数据
- 提炼主题、证据、用户分群、机会和信心等级
- 研究竞品能力、定位、价格、发布节奏和客户反馈
- 生成功能矩阵、定位差异、威胁机会和战略启示

输出研究综合报告和竞品分析简报。

## 步骤 5：指标审查和决策节奏（分析层）

使用 **metrics-tracking** 完成：
- 定义北极星指标、L1 健康指标和 L2 诊断指标
- 对齐 OKR、目标阈值、口径和审查频率
- 识别趋势、异常、数据局限和推荐行动
- 为管理层或团队输出指标记分卡

输出指标框架、仪表盘口径和行动建议。

## 步骤 6：沟通和冲刺落地（输出层）

使用 **stakeholder-comms** 和 **sprint-planning** 完成：
- 按高管、工程、客户、董事会或跨职能伙伴调整沟通口径
- 生成状态更新、发布公告、风险升级或决策记录
- 估算冲刺容量，拆分 P0/P1/P2 和 stretch 范围
- 明确依赖、风险缓解、Definition of Done 和关键日期

输出沟通稿、冲刺计划和执行风险清单。

## 最终输出

将以上步骤整合为完整产品战略包，通常交付：

1. **产品机会分析**：目标用户、问题空间、假设、证据和验证实验
2. **PRD/功能规格**：目标、非目标、用户故事、需求分级、验收标准
3. **路线图和优先级**：RICE/MoSCoW/ICE 评分、MVP 范围、依赖和取舍
4. **研究和竞品简报**：用户洞察、机会、竞品矩阵和战略启示
5. **指标审查**：北极星指标、健康指标、目标、趋势和行动建议
6. **沟通和冲刺计划**：利益相关者更新、风险清单、backlog 和 Definition of Done
