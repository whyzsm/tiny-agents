# Ecommerce Copy Humanizer ZH

把"AI 味很重"的中文电商文案，改成更像真人运营/卖货团队会写的表达，同时保留卖点、转化意图和合规边界。

这不是普通润色器，也不是文学改写器。

它的目标是：**把生硬、模板化、空泛承诺式的中文电商文案，改成更自然、更可信、更像真人写的版本，而且仍然能卖货。**

---

## 速查表

| 场景 | 动作 | 语气建议 | 风险重点 |
|---|---|---|---|
| 商品详情页 | 保留卖点结构，降低 AI 模板感 | 专业克制，像资深运营写的 | 绝对化用语、虚假承诺 |
| 广告/投流素材 | 压缩到 1-2 句核心打动，去掉废话 | 直击痛点，像老板娘喊话 | 夸大疗效、对比贬低竞品 |
| 评论区回复 | 像真人客服在聊天 | 亲切、快速、有温度 | 承诺发货时间/效果 |
| 直播脚本 | 口语化、节奏快、有互动感 | 像朋友推荐，不像念稿 | 限时限量虚假紧迫感 |
| 短视频字幕 | 每句 �j$15 字，信息密度高 | 种草风/测评风/开箱风 | 引流到站外、违规引导 |
| 私信话术 | 简短、回应具体问题 | 像真人客服，不像机器人 | 过度承诺退换货政策 |

## Solves

中文电商文案常见的失败点不是"信息不够"，而是：

- 一眼看出是 AI 写的，语言像模板拼接；
- 语气太硬、太满、太浮夸，降低信任；
- 太像通用营销文案，不像真实商家/运营在说话；
- 改得更自然之后，反而把卖点、CTA、促销重点改没了；
- 平台、客服、直播、短视频、详情页需要的语气完全不同，但文案还在一把梭。

目标：**在"自然表达、转化效率、平台语境、风险控制"之间找到平衡。**

---

## Use when

Use when the user wants to make Chinese ecommerce copy sound less AI-generated and more commercially authentic.

Typical cases:
- 商品详情页文案润色
- 广告文案/投流素材改写
- 评论回复、私信话术优化
- 直播脚本、口播台词、短视频字幕去 AI 味
- 促销文案、活动文案本地化
- 将英文或 AI 初稿改成更自然的中文电商表达
- 让"听起来很会写，但不太会卖"的文案变成更能转化的版本

## Do not use when

Do not use this skill when:
- 用户要的是纯翻译，而不是转化导向改写
- 用户要的是文学创作、品牌 manifesto、长篇散文风格写作
- 用户要的是法律、医疗、金融等高风险专业合规审校
- 输入信息太少，无法判断卖点、场景和风险边界
- 任务本质是从零写完整营销策略，而不是优化已有文案

## Inputs

Ask for the minimum useful context:
- 原始中文文案或 AI 初稿
- 平台 / 场景（详情页、广告、评论区、直播、短信、私信等）
- 产品类型
- 目标受众
- 目标动作（点击、下单、咨询、收藏、回复）
- 语气偏好（自然、克制、种草、强卖点、客服式、直播式）
- 风险边界 / 禁用词 / 禁止承诺

## Workflow

### 1. 诊断 AI 味来源

先识别文案中最明显的 AI 痕迹，按以下清单逐项检查：

**结构类 AI 信号：**
- 每段长度相似、节奏过于整齐
- "首先...其次...最后..." 等过于工整的连接
- 每个卖点都用相同句式展开
- 信息面面俱到但没有重点

**用词类 AI 信号：**（详见 `references/ai-patterns-zh.md`）
- "值得一提的是"、"不仅...而且..."、"为您打造"
- "全方位"、"一站式"、"赋能"、"焕新"
- "让我们一起"、"在这个快节奏的时代"
- "品质之选"、"匠心"、"臻选"等营销空话

**语气类 AI 信号：**
- 语气太满、像 PPT 汇报
- 过度修辞但缺少具体事实
- 承诺太多但没有证据支撑
- 像在写作文而不是在卖货

### 2. 确定文案的商业任务

改写之前，先搞清楚这段文案的"工作"是什么：

| 目标 | 改写方向 |
|---|---|
| 提高点击率 | Hook 要短、狠、直接，激发好奇 |
| 促成下单 | 卖点具体、风险降低、CTA 明确 |
| 降低疑虑 | 加入真实证据、具体数据、退货承诺 |
| 回应异议 | 正面回应、不回避、给解决方案 |
| 评论区互动 | 口语化、短句、像真人在聊天 |
| 直播卖货 | 节奏快、互动多、紧迫感真实 |

### 3. 按场景改写

不同平台有不同的"人味"标准。改写时参照 `references/platform-guide-zh.md` 中的平台语境规则。

核心原则：
- 保留核心卖点、利益点、使用场景、CTA
- 把生硬表达改成更像真人会说的话
- 去掉空泛承诺，换成更具体、更可信的说法
- 保持中文平台语境，不要写成翻译腔
- 根据场景调整语气，不同平台不要混写
- 口语表达 ≠ 随意，仍需保持专业度

### 4. 合规与风险扫描

改写完成后进行风险审查（详见 `references/risk-checklist-zh.md`）：

**高风险词汇：**
- 绝对化：最好、第一、顶级、100%、绝对、完美
- 虚假承诺：保证效果、立刻见效、永久、零风险
- 医疗/功效越界：治疗、治愈、药用级、医疗级
- 对比贬低：比 XX 品牌好、秒杀同类、吊打竞品

**替换建议模式：**
- "最好的" → "我们团队用了 X 年打磨的"
- "100% 有效" → "XX% 的用户反馈有改善"
- "永久保障" → "提供 X 天无理由退换"
- "秒杀竞品" → "在 XX 方面我们做了不同的选择"

### 5. 输出质量自检

在交付改写结果之前，用 `assets/rewrite-checklist.md` 中的清单逐项确认。

## Output

Return a practical rewrite package:

1. **Problem diagnosis**
   - 文案里最明显的 AI 痕迹 / 信任问题 / 转化问题
   - 标注具体的问题句子和原因

2. **Primary rewrite**
   - 最推荐版本，可直接使用或稍作调整
   - 保留原文的核心卖点和商业意图

3. **Alternative rewrite(s)**
   - 更克制版 / 更种草版 / 更转化版（按场景给 1–2 个）
   - 标注每个版本更适合的场景

4. **Risk replacements**
   - 哪些词建议替换，替换成什么更稳
   - 格式：`原文 → 建议 → 原因`

5. **Usage notes**
   - 更适合详情页 / 广告 / 评论区 / 直播哪种场景
   - 是否需要根据不同平台进一步调整

See `references/output-template.md` for full output format.

---

## Quality bar

A strong output should:
- 读起来像真人运营，不像模型模板
- 保留商业目的，而不是只追求"自然"
- 更像真实卖货表达，而不是空洞文案腔
- CTA 清晰，但不过度 push
- 能直接用于发布、回复、投放或脚本改稿
- 明确指出风险边界，而不是盲目"润色"

## What "better" looks like

Good output should feel like:
- 更可信
- 更具体
- 更接近真实中国电商语境
- 更适合平台和场景
- 更容易转化，而不是曵像作文

## Examples

### Before (AI 味重)

> 这款面膜采用先进的玹尿酸科技，为您的肌肤带来全方位的深层滋润。不仅能有效锁住水分，更能让您在忙碌的生活中轻Ꝿ拥有水润透亮的肌肤。品质之选，值得信赖！

**问题：** "全方位"、"不仅...更能"、"品质之选"、"值得信赖"都是典型 AI 模板词。

### After (改写版 — 详情页)

> 冬天空调房脸紧绷？这片面膜敷 15 分钟，玻尟酸直接把水按进皮肤里。第二天上妆不卡粉。28 片 / 盒，每天不到 2 块钱。

**为什么好：** 从用户场景切入，给了具体的使用体验和价格锚点，去掉了所有空话。

### After (改写版 — 小红书种草)

> 姐妹们！空调房救命面膜找到了 😭 敷完第二天妆服帖到同事问我用了什么，一盒 28 片日抛用法不心疼。

**为什么好：** 口语化、有社交分享感、用结果说话。
