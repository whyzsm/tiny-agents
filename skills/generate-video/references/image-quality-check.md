# Image Quality Check - Claude による画像品質判定

Nano Banana Pro で生成した画像を Claude が目視判定し、品質を保証します。

---

## 概要

`image-generator.md` の Step 4 で実行される品質判定ロジックです。
2枚の生成画像を Claude が分析し、採用/不採用を判定します。

## 判定フロー

```
2枚の画像を受け取り
    │
    ├── [Step 1] 各画像を個別評価
    │   ├─ 基本品質チェック
    │   ├─ シーン適合性チェック
    │   └─ ブランド整合性チェック
    │
    ├── [Step 2] 判定結果を集計
    │   ├── 両方 OK → Step 3 へ
    │   ├── 1枚 OK → OK を採用
    │   └── 両方 NG → 再生成へ
    │
    └── [Step 3] 2枚から最適なものを選択
        ├─ 比較評価
        └─ 採用画像を決定
```

---

## 判定基準

### 1. 基本品質チェック

| チェック項目 | 合格基準 | NG 例 |
|-------------|---------|------|
| **鮮明度** | ぼやけ・ノイズなし | ピンボケ、過度なノイズ |
| **構図** | バランスが取れている | 偏り、切れ、不自然な配置 |
| **色調** | 自然で一貫性がある | 色むら、不自然な彩度 |
| **アーティファクト** | 生成ノイズなし | 歪み、繰り返しパターン |
| **解像度** | 指定解像度を満たす | 低解像度、アップスケール劣化 |

### 2. シーン適合性チェック

| チェック項目 | 合格基準 | NG 例 |
|-------------|---------|------|
| **主題の一致** | プロンプトの主題を表現 | 無関係な内容 |
| **スタイルの一致** | 指定スタイルに沿っている | スタイル不一致 |
| **用途適合** | シーンの目的に合っている | 動画に不適切な構図 |
| **テキスト品質** | 含まれる場合は読みやすい | 文字化け、不自然なテキスト |

### 3. ブランド整合性チェック

| チェック項目 | 合格基準 | NG 例 |
|-------------|---------|------|
| **カラースキーム** | ブランドカラーと調和 | 色の不一致 |
| **トーン** | プロフェッショナル | カジュアルすぎる、チープ |
| **一貫性** | 他シーンと統一感 | 浮いている |

---

## 評価スコアリング

### スコア定義

| スコア | 判定 | 説明 |
|--------|------|------|
| 5 | Excellent | 完璧、即採用 |
| 4 | Good | 良好、採用可 |
| 3 | Acceptable | 許容範囲、他がなければ採用 |
| 2 | Poor | 問題あり、再生成推奨 |
| 1 | Unacceptable | 使用不可、必ず再生成 |

### 採用閾値

```
採用閾値 = 3（Acceptable以上）
```

### 2枚比較時の選択ロジック

```
両方が閾値以上の場合:
  → スコアが高い方を採用
  → 同スコアの場合は以下で判断:
     1. シーン適合性が高い方
     2. ブランド整合性が高い方
     3. 1枚目を採用（デフォルト）
```

## Claude による判定プロンプト

### 画像評価プロンプト

````text
以下の画像を評価してください。

## 評価対象
- シーン: {scene_name}
- 用途: {scene_purpose}
- 期待スタイル: {expected_style}
- ブランドカラー: {brand_colors}

## 評価基準
1. 基本品質（鮮明度、構図、色調、アーティファクト）
2. シーン適合性（主題、スタイル、用途）
3. ブランド整合性（カラー、トーン、一貫性）

## 出力形式
```json
{
  "score": 1-5,
  "verdict": "OK" | "NG",
  "strengths": ["良い点1", "良い点2"],
  "issues": ["問題点1", "問題点2"],
  "improvement_suggestions": ["改善案1", "改善案2"]
}
```
````

### 2枚比較プロンプト

````text
以下の2枚の画像を比較し、より適切な方を選択してください。

## 評価対象
- シーン: {scene_name}
- 用途: {scene_purpose}

## 画像A の評価
{image_a_evaluation}

## 画像B の評価
{image_b_evaluation}

## 出力形式
```json
{
  "selected": "A" | "B",
  "reason": "選択理由",
  "comparison_notes": "比較の詳細"
}
```
````

## NG 時の再生成指示

### 改善プロンプト生成

```
前回の画像が不採用となりました。

## 問題点
{issues}

## 改善案
{improvement_suggestions}

## 修正プロンプト
以下の点を改善したプロンプトを生成してください:
1. {improvement_1}
2. {improvement_2}
```

### 再生成時の調整戦略

| 問題カテゴリ | 調整戦略 |
|-------------|---------|
| 構図の問題 | `centered composition`, `balanced layout` を追加 |
| 色の問題 | 具体的な色指定（HEX）を追加 |
| スタイル不一致 | スタイル指定をより具体的に |
| テキスト品質 | `no text` を追加、後から重ねる |
| 品質低下 | `high quality`, `4K`, `detailed` を追加 |

## 実行例

### Read ツールで画像を読み込み

```
Claude が Read ツールで画像を読み込む:

Read:
  file_path: "out/assets/generated/intro_1.png"

Read:
  file_path: "out/assets/generated/intro_2.png"
```

### 評価実行

```
画像を評価中...

📊 評価結果:

| 画像 | スコア | 判定 | 主な評価 |
|------|--------|------|---------|
| intro_1 | 4/5 | ✅ OK | 構図良好、色調統一 |
| intro_2 | 3/5 | ✅ OK | やや暗い、構図に偏り |

🎯 選択: intro_1
理由: 全体的なバランスと明るさが優れている
```

### NG 時の出力例

```
📊 評価結果:

| 画像 | スコア | 判定 | 主な問題 |
|------|--------|------|---------|
| cta_1 | 2/5 | ❌ NG | テキストが崩れている |
| cta_2 | 2/5 | ❌ NG | 色が暗すぎる |

⚠️ 両方の画像が基準を満たしていません

改善案:
1. プロンプトに `no text` を追加（テキストは後から重ねる）
2. `bright colors`, `light background` を追加

再生成を実行します... (試行 2/3)
```

---

## 判定のバイパス

### 強制採用

```
ユーザーが「この画像でいい」と指定した場合:
→ 品質判定をスキップして採用
```

### 閾値調整

```
ユーザーが「もっと厳しく」と指定した場合:
→ 採用閾値を 4 に引き上げ

ユーザーが「とりあえずで」と指定した場合:
→ 採用閾値を 2 に引き下げ
```
