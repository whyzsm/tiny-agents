# 汇报材料工作流

## 场景匹配

| 场景 | 推荐框架 | 时间分配 |
|-----|---------|---------|
| 年度/半年度综合汇报 | 现状-问题-对策-成效 | 各 25% |
| 专项工作汇报 | 战略层-执行层-保障层 | 20%/50%/30% |
| 计划/方案汇报 | 目标-举措-保障 | 20%/50%/30% |
| 常规例会汇报 | 工作概况-亮点成效-下步安排 | 30%/40%/30% |

## 各框架详细说明

### 框架一：现状-问题-对策-成效（综合汇报）

```
一、工作推进情况
   （一）主要做法
   （二）取得的阶段性成果

二、存在的困难问题
   （一）外部挑战
   （二）内部短板

三、下步工作举措
   （一）重点任务
   （二）保障措施

四、取得的综合成效（如成效突出可单列）
```

### 框架二：战略层-执行层-保障层（专项汇报）

```
一、战略规划层
   （一）总体目标
   （二）核心思路

二、落地执行层
   （一）重点任务 1
   （二）重点任务 2
   （三）重点任务 3

三、支撑保障层
   （一）组织保障
   （二）资源保障
   （三）机制保障
```

### 框架三：目标-举措-保障（计划汇报）

```
一、目标任务
   （一）总体目标
   （二）阶段目标

二、重点举措
   （一）举措一
   （二）举措二

三、保障机制
   （一）组织机制
   （二）考核机制
```

## 常见问题处理

### Q1: 素材太少，内容空洞
**处理**：
- 向提供者追问具体数据、案例
- 适当合并到上位主题，避免单独成节显得单薄
- 用"持续深化""稳步推进"等过渡语填充

### Q2: 素材太多，重点不突出
**处理**：
- 按领导关注程度排序，核心内容放前
- 次要内容压缩或归入附录
- 同一主题多表述的，保留最有力的一个

### Q3: 多方数据口径不一致
**处理**：
- 优先使用财务/统计数据
- 标注数据来源和口径差异
- 无法统一的在附录说明

### Q4: 存在负面信息如何呈现
**处理**：
- 客观呈现，不回避
- 重点放在"已采取措施"和"预期改善"
- 转化为下一步工作方向

## 标题提炼技巧

| 问题类型 | 原表述 | 优化后 |
|---------|-------|-------|
| 描述性标题 | "关于市场拓展的情况" | "市场拓展实现新突破" |
| 模糊标题 | "技术创新工作" | "核心技术能力显著增强" |
| 堆砌标题 | "完成了什么、做了什么、推进了什么" | "重点任务按期完成率达 95%" |
| 口语化标题 | "效果还不错" | "核心指标超额完成" |

## 动词使用指南

| 类别 | 推荐动词 |
|-----|---------|
| 进展类 | 推进、落实、深化、完成、实现、达到 |
| 成果类 | 突破、构建、打造、形成、取得、荣获 |
| 提升类 | 增强、提高、改善、优化、巩固、提升 |
| 管控类 | 加强、严格、规范、强化、夯实、筑牢 |
