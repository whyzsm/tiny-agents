# HarmonyOS 专家团

`$harmony-expert-team` HarmonyOS/OpenHarmony 项目专家团入口

## 触发的技能

| 技能 | 能力 |
|---|---|
| `$harmony-expert-team` | HarmonyOS/OpenHarmony 专家团工作流总入口 |
| `$harmony-os-ask` | ArkTS、ArkUI、DevEco Studio、hvigor、配置和错误诊断问答 |
| `$harmony-os-act` | HarmonyOS 项目实现、修复、重构、路由配置、权限和构建验证 |
| `$generate-ui-code` | 从需求、截图、草图或设计稿生成 ArkUI 页面和组件 |
| `$service-widget` | 服务卡片、元服务卡片、form_config.json、卡片资源和预览调试 |

## 我可以帮你做这些

1. HarmonyOS 技术问答

   调用 `harmony-os-ask`

   解释 ArkTS、ArkUI、生命周期、项目配置、构建错误和 API 使用方式。

2. 项目实现和修复

   调用 `harmony-os-act`

   在现有 HarmonyOS 项目中实现功能、修复 bug、调整路由、配置权限并执行可用验证。

3. ArkUI 页面生成

   调用 `generate-ui-code`

   根据产品需求、截图、草图或设计规范生成可落地的 `.ets` 页面和组件。

4. 服务卡片开发

   调用 `service-widget`

   创建或调试服务卡片、元服务卡片、卡片配置、资源引用和预览保存流程。

5. 端到端 Harmony 交付

   调用 `harmony-os-ask`、`harmony-os-act`、`generate-ui-code`、`service-widget`

   先诊断需求和项目结构，再完成代码实现、UI 生成、卡片开发和构建检查。

## 完整交付物通常是

```text
HarmonyOS 技术诊断和方案说明

ArkTS / ArkUI 代码实现或修复

页面、组件、路由和资源配置

服务卡片或元服务卡片配置与代码

构建、静态检查或可行验证结果

变更文件、风险和后续建议
```
