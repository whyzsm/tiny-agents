# AGC 实测校验流程

这份流程用于得到华为 AppGallery Connect 控制台的真实“上架自检”结果。
它不是本地扫描的替代品，也不把本地结果伪装成 AGC 结果。

## 结果边界

AGC 的“软件包管理”页面至少有两个容易混淆的状态列：

- **合法性**：表示上传包已被 AGC 接受为合法包，例如 `已达标`。这不等于
  上架自检通过。
- **上架自检**：表示云测试的状态，例如 `检测中`、`待优化` 或 `已达标`。
  只有这一列的 `已达标` 才能记录为 `AGC_READY`。

报告页还会显示测试结果、报告 ID、测试时间、测试设备、API Level、测试场景
和应用大小，并按以下维度展开：

| AGC 维度 | 必须记录 |
|---|---|
| 兼容性 | 测试总项、不通过项、警告项、通过项、设备和系统版本覆盖 |
| 稳定性 | 测试总项、不通过项、警告项、通过项、影响机型和执行失败说明 |
| 功耗 | 测试总项、不通过项、警告项、通过项、设备和系统版本 |
| 性能 | 测试总项、不通过项、警告项、通过项、设备和系统版本 |
| UX | 测试总项、不通过项、警告项、通过项、问题名称、影响机型和说明文档 |

AGC 可能出现“测试结果 = 执行失败”，即使某些维度已经显示通过率或警告。
这种情况必须记录为 `AGC_EXECUTION_FAILED`，不能降级成通过。

## 浏览器执行步骤

1. 在已登录且拥有目标应用权限的 AGC 页面打开目标应用的
   `应用上架 > 软件包管理`。
2. 先在本地运行静态预检，确认待测包的文件名、包名、版本名、版本号和
   release 配置。只选择这一份明确的 `.app` 或 `.hap`。
3. 检查软件包行的包名、版本、设备类型和使用场景，确保与目标发布版本
   一致。不要把同包名的历史上传记录当成当前包。
4. 若需要上传，先向用户确认“将把指定发布包上传到华为 AGC 的哪个账号/应用”。
   用户确认后才上传。不要上传证书、Profile、密码或无关文件。
5. 等待 `合法性` 结果。若不是 `已达标`，先处理包合法性问题，不要启动
   上架自检。
6. 在同一行点击 `启动自检`。测试设备范围必须覆盖 listing 中声明的设备类型；
   如果只测了手机而应用声明了平板/折叠屏，报告只能记为覆盖不完整。
7. 轮询到 `上架自检` 不再是 `检测中` 后打开 `报告`，读取总览和五个维度。
   测试仍在运行、页面执行失败或报告缺维度时，不得声称已通过。
8. 把报告 ID、时间、包身份、API Level、设备覆盖和每个维度的统计写入最终
   报告，并保留 AGC 报告链接作为证据。

## 归档格式

最终报告可按以下结构记录。字段值必须来自 AGC 页面或明确标注为未验证：

```json
{
  "source": "agc-console",
  "report_id": "<AGC report id>",
  "agc_status": "已达标 | 待优化 | 检测中 | 执行失败",
  "test_result": "<AGC 测试结果>",
  "package": {
    "file_name": "<artifact file name>",
    "bundle_name": "<bundle name>",
    "version_name": "<version name>",
    "version_code": "<version code>"
  },
  "api_level": "<API level>",
  "tested_devices": ["<model and system version>"],
  "device_coverage_complete": false,
  "categories": {
    "兼容性": {"total": 0, "failed": 0, "warnings": 0, "passed": 0},
    "稳定性": {"total": 0, "failed": 0, "warnings": 0, "passed": 0},
    "功耗": {"total": 0, "failed": 0, "warnings": 0, "passed": 0},
    "性能": {"total": 0, "failed": 0, "warnings": 0, "passed": 0},
    "UX": {"total": 0, "failed": 0, "warnings": 0, "passed": 0}
  }
}
```

## 判定规则

| AGC 页面状态 | Skill 结果 | 说明 |
|---|---|---|
| 上架自检 `已达标` | `AGC_READY` | AGC 实测完成且页面明确达标；仍不能代替人工审核承诺 |
| 上架自检 `待优化` | `AGC_NOT_READY` | 存在 AGC 优化项，先处理报告 |
| 上架自检 `检测中` | `AGC_PENDING` | 测试未完成，不给通过结论 |
| 测试结果 `执行失败` | `AGC_EXECUTION_FAILED` | 测试未完整执行，不是通过 |
| 无登录、无报告、缺少维度或设备覆盖不明 | `UNVERIFIED` | 补充 AGC 证据后再判定 |

## 安全边界

- 页面中出现的账号、应用 ID、报告 ID、包名和设备信息可作为结果证据，
  但不要把密码、Cookie、token、证书或 Profile 写入报告。
- 只读取报告和状态。删除软件包、修改 listing、提交审核和发布都不属于
  自检，必须另行确认。
- AGC 控制台的当前规则高于本仓库中的基线矩阵；页面没有显示的规则不要
  自行补成“官方要求”。
