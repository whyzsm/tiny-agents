# HarmonyOS AppGallery 自检矩阵

这份矩阵是 `$harmonyos-app-store-self-check` 的工作资料，不是华为应用市场的官方承诺或法律意见。它把输入资料中可见的“上架护航”内容整理成可验证的证据项；对于审核规则、地区要求、当前 API/AGC 字段和账号能力，必须以当前官方文档或 AGC 控制台为准。

## 证据状态

| 状态 | 含义 | 例子 |
|---|---|---|
| `PASS` | 已有可复核证据，且没有发现阻断项 | 脚本通过、release 构建成功、真机流程记录 |
| `FAIL` | 已发现不符合项目或发布门禁的事实 | 公开仓库含密码、声明了未使用的敏感权限 |
| `BLOCKED` | 需要用户、AGC 或真实设备才能继续 | 缺少签名材料、未登录 AGC、无可用真机 |
| `UNVERIFIED` | 尚未检查，不能推断结果 | 截图未提供、审核政策未确认 |

## 检查矩阵

### 1. 工程与身份

| ID | 检查 | 证据 | 失败影响 |
|---|---|---|---|
| `META-1` | AppScope/app.json5 存在且有 bundleName、label、versionCode、versionName | 文件路径、字段值、资源引用 | `P0`：包身份或版本不可确认 |
| `META-2` | module.json5 有 mainElement、pages/路由注册、deviceTypes | 模块配置和 main_pages.json | `P1`：安装后可能无法启动或缺页 |
| `META-3` | target/compatible/compile SDK 与目标设备、AGC 支持范围相符 | build-profile.json5、官方版本资料 | `P1`：兼容性或上传校验失败 |
| `META-4` | 应用名称、图标、描述、版本与待上传 artifact 一致 | listing 文案、资源、artifact 元数据 | `P1`：信息不一致或误导审核 |

### 2. 隐私、权限与数据

| ID | 检查 | 证据 | 失败影响 |
|---|---|---|---|
| `PRIV-1` | 隐私政策可访问，说明收集范围、用途、存储、删除/清除、联系方式和更新方式 | 应用内/市场 URL、法律文件 | `P0`/`P1`：隐私材料缺失或不完整 |
| `PRIV-2` | 权限声明、运行时申请、用户可见说明与真实功能一致 | module.json5、Ability 代码、页面文案、隐私政策 | `P0`：越权或用途不透明 |
| `PRIV-3` | 权限拒绝、取消、不可用和数据清除路径有可见反馈 | 真机/模拟器记录、页面状态 | `P1`：用户无法理解或撤回处理 |
| `DATA-1` | 敏感数据不以明文或不必要的远端路径保存 | SQLite schema、媒体层、存储策略 | `P1`：泄露或合规风险 |
| `DATA-2` | 删除业务记录会同步处理派生索引和本地媒体 | repository、cleanup service、回归测试 | `P0`：用户无法真正删除隐私数据 |
| `DATA-3` | 第三方 SDK、网络、广告、支付、账号、儿童/未成年人、健康和位置能力按实际情况披露 | 依赖、源码、隐私政策、listing | `P0`/`P1`：材料与行为不一致 |

### 3. 签名、构建与发布包

| ID | 检查 | 证据 | 失败影响 |
|---|---|---|---|
| `SIGN-1` | 产物是目标 product 的 release HAP/APP，不是 debug 包 | hvigor 命令、产物路径、构建日志 | `P0`：无法作为发布包使用 |
| `SIGN-2` | AGC 证书、profile、keystore 与 bundle/权限/版本匹配 | AGC 或签名工具输出（只保留摘要） | `P0`：上传或安装校验失败 |
| `SIGN-3` | 证书、profile、p12、密码、token、home 路径未进入提交内容 | `git ls-files`、暂存区扫描、`.gitignore` | `P0`：密钥泄露，必须停止发布 |
| `SIGN-4` | release 构建可重复，依赖已锁定且无未说明本地模块 | lockfile、ohpm、hvigor 日志 | `P1`：构建不可复现 |

### 4. 功能、稳定性与体验

| ID | 检查 | 证据 | 失败影响 |
|---|---|---|---|
| `QA-1` | 冷启动、首次运行、升级/迁移、重启回显和异常恢复通过 | 手工 QA/自动化日志 | `P0`/`P1`：核心流程阻断或数据丢失 |
| `QA-2` | 核心 CRUD、导航、空/加载/错误/禁用状态可达且反馈清楚 | 页面路径、测试记录 | `P1`：审核演示或用户使用失败 |
| `QA-3` | 无障碍名称、触达区、长文本、屏幕/折叠/平板适配经过适用范围验证 | 设备截图、无障碍检查 | `P1`/`P2`：可用性风险 |
| `QA-4` | 没有虚构远端同步、作者、点赞、评论、推荐或未实现能力 | 页面文案、静态数据、截图 | `P0`：产品描述和真实能力不一致 |

### 5. 应用市场资料

| ID | 检查 | 证据 | 失败影响 |
|---|---|---|---|
| `LIST-1` | 图标、截图、视频（如有）来自真实可达状态，无调试 UI、占位和敏感信息 | 资源、截图、录屏 | `P1`：资料不合格或误导 |
| `LIST-2` | 描述、分类、支持设备、隐私 URL、开发者信息与实现一致 | listing 草稿和项目配置 | `P1`：审核驳回风险 |
| `LIST-3` | 版权、商标、第三方图片/字体/SDK 使用权可说明 | 资产清单、许可文件 | `P1`：侵权或材料缺失 |

## Workshop 可见资料映射

外部 `harmonyos-code-workshop` 可见章节提供了以下基线：

- 发布顺序：开发者注册 → AGC 创建应用 → 准备证书/profile → 签名打包 HAP/APP → 填写国家/地区、价格、隐私政策链接 → 提交审核。
- 安全合规：最小权限、敏感权限用途说明、隐私政策、应用描述与功能一致、涉及未成年人时提供相应保护。
- 构建参考：release HAP/APP 命令、CI 中不依赖 daemon、签名密码通过环境变量注入。
- 签名材料：`.p12`、`.csr`、`.cer`、`.p7b` 及 profile 的职责区分。

这些内容没有覆盖完整的当前 AGC 字段、地区法规、内容审核规则、账号资质或尺寸阈值。报告中必须标记这些缺口，不要把“资料里没有提到”写成“官方不要求”。

## Harmony Wardrobe 证据地图

识别到本项目（`io.wardrobe.tiny` / `衣不缺`）时，追加以下门禁：

1. 离线启动、保存、重启回显应在无网络状态下验证；当前产品不应声明网络同步，也不应新增 `INTERNET` 权限。
2. 衣物、穿搭、穿着记录、逛店和心愿数据写入本地 SQLite；照片先经媒体层复制到应用本地目录，数据库只保留本地 URI/path。
3. 删除记录必须同时验证数据库行、搜索索引和本地照片清理；失败时给用户可见错误，不得静默保留外部 URI。
4. 隐私政策和用户协议位于 `docs/legal/`；应用市场截图位于 `docs/appgallery/screenshots/`，只能作为当前版本证据，不自动等同于 AGC 已接受。
5. `build-profile.json5` 中的本机 `signingConfigs`、密码、证书/profile 和 `/Users/` 路径只能作为本地发布输入，不能进入仓库提交或共享报告。
6. 若需修改 UI，先读取 `docs/background/yibuque-design.md`；上架自检不应顺带改变产品视觉方向。

## 报告模板

```text
结论：BLOCKED
检查范围：<product/module/API/commit/artifact>

| ID | 状态 | 严重度 | 证据 | 处理 |
| SIGN-3 | FAIL | P0 | build-profile.json5: signing material | 移除提交内容，改用发布环境注入 |
| PRIV-1 | UNVERIFIED | P1 | 未提供 AGC 隐私 URL | 补充 URL 并在 AGC 实际检查 |

已运行：<commands>
未验证：<device/AGC/account checks>
下一步：<one smallest action>
```
