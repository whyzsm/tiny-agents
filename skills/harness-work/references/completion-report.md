# Completion Report

The Lead writes the completion report after review passes.

## Solo / Single Task

Include:

- task id and title
- what changed
- user-visible before / after
- changed files
- validation commands
- review verdict
- commit hash when available
- remaining `Plans.md` work

## Breezing / Multi Task

Collect:

```bash
git log --oneline "${BASE_REF}..HEAD"
git diff --stat "${BASE_REF}..HEAD"
```

Then report:

- completed task count
- each task title with commit hash
- total changed files and diff stat
- remaining `cc:TODO` / `cc:WIP` tasks
- failed or blocked tasks, if any

The report is generated by the Lead, not by a Worker or hook.
