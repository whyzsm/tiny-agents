# 合同智能助手 v1.0.3

## 技能概述

合同智能助手是一款全能型合同管理工具，集合同生成、审查、管理于一体。支持从合同起草到归档的全流程管理，包括智能生成、风险识别、合规审查、条款对比、修改建议等专业功能。适用于企业法务、个人签约、合同管理等场景。

## 📋 分阶段工作流程

### 阶段一：合同生成（新增）

#### 1.1 模板选择模式

**输入**：用户选择合同类型

**处理**：
1. 展示对应类型的合同模板列表
2. 用户选择模板或选择AI智能起草
3. 进入信息填写环节

**输出**：
```
📋 合同类型：劳动合同
可选用模板：
1. 标准劳动合同模板（通用）
2. 高管劳动合同模板（含竞业限制）
3. 技术岗位劳动合同模板（含保密条款）
4. 销售岗位劳动合同模板（含提成条款）
5. 🤖 AI智能起草（根据需求定制）
```

#### 1.2 对话引导填写

**输入**：用户选择模板

**处理**：
1. 逐项询问合同关键信息
2. 智能提示必填项和可选项
3. 实时验证信息合法性

**输出**：
```
🤝 让我们开始填写合同信息

第1步（共8步）：合同基本信息
├─ 甲方（用人单位）名称：？
├─ 统一社会信用代码：？
├─ 法定代表人：？
└─ 注册地址：？

提示：可上传营业执照自动填写 ↗
```

#### 1.3 AI智能起草

**输入**：用户描述合同需求

**处理**：
1. 分析合同目的和核心条款
2. 智能生成合同框架
3. 自动填充常见条款
4. 标注需人工确认的关键条款

**输出**：
```
📄 AI起草进度
├─ 合同框架生成 ✓
├─ 必备条款填充 ✓
├─ 风险条款规避 ✓
├─ 合规性检查 ✓
└─ ⚠️ 需确认：竞业限制期限建议24个月，是否调整？
```

#### 1.4 条款建议

**输入**：特定场景或条款类型

**处理**：
1. 根据合同类型推荐关键条款
2. 提供多版本条款备选
3. 说明条款风险和适用场景

**输出**：
```
💡 条款建议：保密条款

【推荐版本】（平衡保护）
双方确认，乙方在任职期间接触的甲方商业秘密包括但不限于：
客户名单、技术方案、财务数据等。保密期限为离职后2年。

【强保护版本】（适合高管）
除上述内容外，还包括：经营策略、供应商信息等。
违约金设置为：离职前12个月平均工资的3倍。

【简化版本】（适合普通岗位）
仅限于：技术文档、客户信息。
无违约金约定，承担法定保密义务。
```

#### 1.5 Word导出

**输入**：审核完成的合同内容

**处理**：
1. 格式化为标准合同格式
2. 添加编号、签章区
3. 生成Word文档

**输出**：
```
📄 合同文档已生成

文件名：劳动合同_张三_20260425.docx
大小：128KB
页数：8页

包含内容：
├─ 合同正文 ✓
├─ 附件清单 ✓
├─ 签章区 ✓
└─ 版本信息 ✓

📤 [下载Word文档]
```

#### 1.6 合同存档

**输入**：签署完成的合同

**处理**：
1. 合同分类归档
2. 记录合同版本历史
3. 设置提醒事项（到期提醒等）

**输出**：
```
📁 合同已归档

归档路径：/劳动合同/深圳市XX科技有限公司/
├─ 员工姓名：张三
├─ 合同编号：HT-2026-04-025
├─ 签订日期：2026-04-25
├─ 到期日期：2027-04-24
├─ 版本：v1.0（首次签订）
└─ ⏰ 已设置：到期前30天提醒
```

---

### 阶段二：合同接收（原有）

**输入**：用户上传的合同文本（完整条款或关键段落）

**处理**：
1. 接收并解析合同文本
2. 自动识别合同类型（劳动合同/买卖合同/租赁合同/服务合同等）
3. 提取合同基本信息（甲方、乙方、签订日期、标的等）

**输出**：
```
{
  "合同类型": "劳动合同",
  "甲方": "XX科技有限公司",
  "乙方": "张三",
  "合同期限": "2024.01.01 - 2025.12.31",
  "关键条款数": 12
}
```

**关键检查点**：⚠️ 确认合同文本完整，无缺失章节

---

### 阶段三：条款提取（原有）

**输入**：合同文本

**处理**：
1. 按条款类型分类提取
2. 识别关键条款要素（标的/价款/期限/违约责任等）
3. 标注条款编号便于引用

**输出**：
```
{
  "标的条款": "第3条：乙方负责XXX工作...",
  "价款条款": "第5条：月薪25000元...",
  "保密条款": "第8条：乙方需遵守保密义务...",
  "解除条款": "第12条：双方可依法解除合同..."
}
```

**关键检查点**：⚠️ 检查条款编号连续性，识别缺失条款

---

### 阶段四：风险扫描（原有）

**输入**：提取的条款

**处理**：
1. 逐条风险评估（高/中/低风险）
2. 识别显性风险（明确不利条款）
3. 识别隐性风险（模糊表述/灰色地带）
4. 标记需特别注意的条款

**输出**：
```
{
  "高风险条款": [
    {"条款": "第7条", "风险": "竞业限制范围过宽", "建议": "限定行业范围和地域"}
  ],
  "中风险条款": [...],
  "低风险条款": [...]
}
```

**关键检查点**：⚠️ 特别注意违约金、定金、免责条款

---

### 阶段五：合规检查（原有）

**输入**：合同文本 + 适用法规

**处理**：
1. 核对法律强制性规定
2. 验证格式条款有效性
3. 检查管辖约定合法性
4. 验证时效条款合规性

**输出**：
```
{
  "合规问题": [
    {"条款": "第15条", "问题": "约定管辖违反级别管辖", "依据": "《民事诉讼法》"}
  ],
  "格式条款审查": [...],
  "效力待定条款": [...]
}
```

**关键检查点**：⚠️ 劳动法对劳动者倾斜保护的规定必须遵守

---

### 阶段六：报告生成（原有）

**输入**：风险扫描结果 + 合规检查结果

**处理**：
1. 综合风险评级（高危/中危/低危）
2. 生成结构化审查报告
3. 提供具体修改建议

**输出**：完整的合同审查报告

### 案例一：劳动合同审查

**【输入示例】**

```
劳动合同（摘要）
甲方：深圳市XX科技有限公司
乙方：李四
签订日期：2024年3月15日

第三条 工作内容和地点
乙方同意在技术部门担任高级工程师，工作地点为深圳市南山区。

第五条 薪酬待遇
乙方的月工资为税前28000元，每月15日前发放。

第七条 竞业限制
```

## ⚠️ 避坑提示

### 一、常见错误清单

#### 1. 忽略主体资格审查 ❌

**反面案例**：
```
合同相对方为"XX网络科技有限公司"，但未核实其是否实际存续。
签约后发现该公司为皮包公司，无财产可供执行。
```

**正确做法** ✅：
- 要求提供营业执照复印件或通过国家企业信用信息公示系统核实
- 核实签约代表是否有合法授权
- 大额合同建议进行尽职调查

#### 2. 遗漏隐蔽条款 ❌

**反面案例**：
```
劳动合同仅约定月薪25000元，未注意到附件包含绩效考核方案。
实际月收入因绩效考核可能降至15000元。
```

**正确做法** ✅：
- 要求提供合同附件并纳入合同整体审查
- 注意"本合同附件与本合同具有同等效力"条款
- 审查任何"以附件为准"、"甲方享有解释权"等兜底条款

#### 3. 口头承诺未书面化 ❌

**反面案例**：
```
销售经理口头承诺"保修期延长至3年"，合同仅写"按国家规定执行"。
保修期满后，销售经理已离职，无法证明口头承诺。
```

**正确做法** ✅：
- 所有承诺必须写入合同或作为附件
- 补充协议需双方签章确认
- 保留谈判过程中的往来函件

---

### 二、典型反面案例

#### 1. 违约金比例过高的反面案例

**场景**：技术合同约定"任何一方违约，需向对方支付合同总价200%的违约金"

**问题分析**：
- 法律允许的违约金以实际损失为限
- 过高的违约金约定可能被法院/仲裁机构调减
- 过高违约金反而助长恶意违约方拖延谈判

**正确做法** ✅：
```python
# 建议违约金比例
违约场景 = {
    "逾期付款": "中国人民银行同期贷款利率的1.3-1.5倍",
    "逾期交货/交付成果": "每日0.03%-0.05%（年化11%-18%)",
    "根本违约": "合同总价的10%-30%",
    "侵犯知识产权": "合同总价或损失额的1-3倍"
}
```

#### 2. 定金约定不规范的反面案例

**反面**：
```
合同约定："买方需在签约时支付定金5万元，余款于收货后支付。"
（仅约定了定金数额，未约定定金罚则）
```

**问题**：未明确约定定金罚则，一旦发生争议，难以主张定金权利

**正确做法** ✅：
```python
# 规范的定金条款示例
定金条款 = """
第X条 定金
1. 买方应于本合同签订之日起X日内向卖方支付定金人民币X元。
2. 若买方违约，卖方有权没收全部定金。
3. 若卖方违约，应当向买方双倍返还定金。
4. 定金作为合同价款的一部分，在最后一期付款时抵扣。
"""
```

#### 3. 格式条款的陷阱

**反面案例**：
```
某APP用户协议约定："用户理解并同意，平台有权随时修改服务内容，
无需通知用户，用户继续使用服务视为接受修改。"
```

**问题**：
- 剥夺用户知情权
- 加重用户义务
- 可能构成无效的格式条款

**法律依据**：
```python
# 《民法典》第497条 格式条款无效情形
格式条款无效 = [
    "具有本法第一编第六章第三节和本法第五百零六条规定的无效情形",
    "提供格式条款一方不合理地免除或者减轻其责任、加重对方责任",
    "提供格式条款一方限制、排除对方主要权利"
]
```

## 🔧 功能实现说明

### LLM智能推理功能（核心能力）

以下功能依赖大语言模型进行智能分析：

| 功能 | 实现方式 | 说明 |
|------|----------|------|
| 合同类型识别 | LLM推理 | 根据文本特征识别合同类型 |
| 条款语义分析 | LLM推理 | 理解条款含义，识别潜在风险 |
| 风险等级评估 | LLM推理 | 综合判断风险严重程度 |
| 条款关联分析 | LLM推理 | 识别跨条款的关联风险 |
| 修改建议生成 | LLM推理 | 基于风险分析生成专业建议 |
| 法律依据匹配 | LLM推理+知识库 | 关联相关法条 |
| 合同智能生成 | LLM推理 | 根据需求自动生成合同 |
| 条款智能建议 | LLM推理 | 针对场景提供专业条款 |

### 需要外部数据支持的功能

| 功能 | 外部依赖 | 建议数据源 |
|------|----------|------------|
| 企业工商信息查询 | 需联网查询 | 国家企业信用信息公示系统 |
| 法规数据库更新 | 需定期同步 | 北大法宝/法信等 |
| 裁判文书参考 | 需联网查询 | 中国裁判文书网 |
| 合同模板库 | 本地存储 | 可扩展的模板库 |
| Word文档生成 | 文档处理库 | python-docx |

### 已知局限性

1. **知识截止日期**：模型知识有截止日期，最新法规变化可能未包含
2. **地域性差异**：不同地区可能有地方性法规差异
3. **专业复杂度**：极专业的法律问题建议咨询执业律师
4. **非诉业务**：本技能侧重合同生成与审查，不提供诉讼代理服务
