# OneQuant 4.0 常见问题 FAQ（28 问）

> 本文档是 SKILL.md 常见问题章节的完整版，包含 28 个常见问题及解答。

---

**Q1: 为什么选股结果为空？**
A: 选股依赖实时行情，在非交易时段（周末、节假日、收盘后）运行可能返回空结果。请在交易时段内（9:30-15:00）使用，或改用回测模块。

---

**Q2: JQData/RQData 测试连接失败怎么办？**
A: 首先确认账号已登录且有有效 license。在「系统设置」→「API 调试」中查看具体错误信息。常见原因：`pip install jqdatasdk` 或 `pip install rqdatac` 未安装。

---

**Q3: K 线图显示异常价格（如 80 元 vs 实际 13 元）？**
A: v4.0 已修复此问题。根因是切换不同股价的股票时 ECharts 缓存未清除。如果仍出现，刷新页面即可。

---

**Q4: 分钟级 K 线数据量很少？**
A: AkShare `stock_zh_a_minute()` 接口约返回近 5 个交易日的数据，非全量历史。如需更长历史分钟数据，请使用 JQData/RQData。

---

**Q5: 策略保存失败（400 错误）？**
A: 系统在沙箱环境中运行时会自动使用 workspace 可写目录。如果在普通环境中，检查 `strategies.json` 是否可写。用管理员身份启动 Flask 可解决权限问题。

---

**Q6: 可以同时使用多个数据源吗？**
A: 可以。在 Web 界面中切换数据源不需重启服务。`dual_adapter.py` 支持双数据源融合模式（一个为主，另一个降级备用）。

---

**Q7: 实盘交易怎么配置？**
A: 实盘模块默认使用 `mock_broker` 模拟券商。对接真实券商需配置对应的 adapter（华泰 miniQMT / 同花顺 iFinD / 东方财富掘金），并提供有效的账号凭证。实盘下单操作需用户明确确认。

---

**Q8: 如何修改 Flask 端口（默认 5001 被占用）？**
A: 修改 `run.py` 最后一行的 `port=5001` 为其他端口（如 8080），然后重启 Flask。或者设置环境变量 `set FLASK_PORT=8080`（Windows）后启动。

---

**Q9: 回测结果和实盘差距很大？**
A: 回测默认不含交易成本（佣金、印花税、滑点）。实盘时需自行折算：A 股佣金约 0.025%（双边），印花税 0.1%（卖出），滑点按成交金额 0.1~0.5% 估算。

---

**Q10: 如何备份和恢复数据？**
A: 备份：复制 `F:\OneQuant_4.0\` 下的 `strategies.json`、`trades.json`、`watchlist.json`、`backend\data\` 目录。恢复：将备份文件放回原路径，重启 Flask 即可。

---

**Q11: 模拟账户的钱是真实的吗？**
A: 不是。模拟账户初始资金 100,000 元是虚拟的，所有交易不产生真实资金变动。模拟交易仅用于验证策略和练习操作。

---

**Q12: 为什么我的委托没有成交？**
A: 模拟交易使用限价单，只有当市场价格触及委托价时才会成交。如果股价快速跳过委托价，可能导致不成交。建议改用市价单。

---

**Q13: 如何批量导出所有数据？**
A: Web 界面「数据导出」Tab 提供一键导出：交易记录、持仓清单、回测报告、策略列表、自选股，均支持 Excel 格式。也可以使用 CLI：`python scripts\cli.py export all`。

---

**Q14: 系统运行变慢怎么办？**
A: 长期运行后 `trades.json` 和回测记录可能过大。建议：1) 定期导出并清理历史回测记录；2) 重启 Flask 释放内存；3) 检查是否有大量未成交委托（可批量撤销）。

---

**Q15: 为什么数据源测试显示"超时"？**
A: AkShare 请求超过 8 秒会自动返回 Mock 数据（降级机制）。如果频繁超时，检查网络连接，或切换到 JQData/RQData（付费数据源更稳定）。

---

**Q16: Flask 启动后无法访问（浏览器显示无法连接）？**
A: 请按以下步骤排查：1) 确认 Flask 进程正在运行（PowerShell 执行 `netstat -ano | Select-String ":5001"`）；2) 确认防火墙未阻止端口 5001；3) 尝试用 `localhost:5001` 代替 `127.0.0.1:5001`；4) 查看 Flask 终端输出是否有报错信息。

---

**Q17: API 返回 `{"success": false}` 如何排查？**
A: 检查返回的 `message` 字段（具体错误原因）、`error_code` 字段（错误代码）、`suggestion` 字段（修复建议）。常见错误：股票代码格式错误（需要 6 位数字）、数据源未配置、券商未连接。

---

**Q18: 如何添加自定义策略（非 MA/RSI/MACD）？**
A: 当前版本支持 MA/RSI/MACD 三种内置策略。添加自定义策略需要修改 `run.py` 中的回测逻辑（参考 `/api/backtest/ma` 实现）。计划在 v4.1 中开放策略插件接口。

---

**Q19: 回测结果与预期不符（收益差异大）？**
A: 请检查：1) 回测区间是否包含完整的牛熊周期；2) 是否考虑了交易成本（佣金、印花税、滑点）；3) 策略参数是否过拟合（针对特定股票调优）；4) 数据质量（AkShare 免费数据可能有缺失）。

---

**Q20: 如何从 v3.03 迁移数据到 v4.0？**
A: v4.0 的数据文件格式与 v3.03 兼容。直接复制 `strategies.json`、`trades.json`、`watchlist.json` 到 v4.0 目录即可。首次启动 v4.0 时会自动迁移。

---

**Q21: 系统要求是什么？**
A: **最低配置**：Windows 10/11，Python 3.8+，4GB RAM，500MB 磁盘空间。**推荐配置**：Python 3.11+，8GB RAM，SSD 硬盘。**不支持**：macOS、Linux（需要自行修改路径相关代码）。

---

**Q22: 如何更新到最新版本？**
A: 1) 备份当前数据文件（`strategies.json`、`trades.json`）；2) 下载最新版本 zip 并解压到新目录；3) 复制备份的数据文件到新目录；4) 启动新版本验证数据正常。

---

**Q23: 发现 Bug 或有功能建议如何反馈？**
A: 1) 在 SkillHub 平台该 Skill 页面下方「评论」区留言；2) 发送邮件到 onquant-support@example.com；3) 在 GitHub 仓库提交 Issue（如果有开源仓库）。

---

**Q24: 为什么模拟交易没有滑点模拟？**
A: 当前版本模拟交易使用限价单撮合，未模拟滑点。实盘交易滑点约为成交金额的 0.1~0.5%。如需更真实的模拟，建议在回测收益中手动扣除滑点成本。

---

**Q25: 可以同时登录多个券商账号吗？**
A: 当前版本仅支持同时连接一个券商。如需多账号管理，需要启动多个 Flask 实例（不同端口），每个实例连接不同的券商账号。

---

**Q26: 数据文件损坏或丢失如何恢复？**
A: 如果有备份，直接恢复备份文件。如果没有备份，系统会在下次启动时自动初始化默认数据（3 个示例策略）。历史回测记录和交易记录无法恢复，建议定期导出备份。

---

**Q27: 为什么选股结果中有些股票无法交易（停牌/退市）？**
A: 选股模块基于最新行情数据筛选，可能包含当天停牌的股票。建议在选股结果中二次过滤：调用 `/api/stock/info` 检查 `status` 字段，排除停牌股票。

---

**Q28: 可以在无网络环境下使用吗？**
A: 部分功能可以：1) 模拟交易（无需网络）；2) 查看本地缓存的 K 线数据；3) 查看历史回测记录。但需要网络的功能：实时行情、实时选股、数据源测试。建议在有网络环境下使用，或提前缓存数据。

## 📋 新增问题（针对 TRACE 评测反馈）

**Q29: 这个工具和同花顺 iFinD / 通达信有什么区别？为什么选它？**
A: 核心差异：①OneQuant 完全本地运行，数据不出本机，无需注册云账号；②10² 个 API 全部开放，可以自由组合成自动化工作流（同花顺/通达信的 API 需额外付费且功能受限）；③支持 AkShare 免费数据源，零成本起步；④从模拟交易到实盘迁移使用同一套接口，无需重写代码。⑤源码完全可见可改。详见 SKILL.md「独家亮点」章节的竞品对比表。

---

**Q30: 数据准确吗？回测结果能信吗？**
A: 诚实说明：
- 日线收盘价：精确（来自交易所官方数据）
- 分钟线：AkShare 仅近 5 天，JQData/RQData 有全量历史
- 回测收益：默认不含佣金/滑点/印花税，实盘需自行扣除约 0.3-0.6% 成本
- 买卖点信号：基于日线收盘价计算，盘中实时信号有确认延迟
- 选股结果：基于全市场快照数据，涨停判断准确，但可能包含停牌股
详见 SKILL.md「精度保证矩阵」（9 项功能 × 2 种数据源的精度标注）。

---

**Q31: 只能用网页操作吗？能不能批量/自动运行？**
A: 支持 5 种调用方式：
① Web 界面（浏览器点点点）② REST API（Python requests 调用）③ CLI 命令行 ④ 批量脚本（多股票并行）⑤ Docker 容器化
定时自动回测示例：配合 Windows 计划任务，每天 15:30 自动对自选股跑 MA 策略回测并生成报告。详见 SKILL.md「调用方式总览」和「批量调用示例」。

---

**Q32: 系统崩溃过吗？数据会丢失吗？**
A: 设计了三层容错机制确保永不崩溃：
第 1 层 — AkShare 请求超过 8 秒自动返回 Mock 降级数据（不白屏）
第 2 层 — 主数据源失败时自动切换到备用数据源
第 3 层 — 所有核心数据持久化为本地 JSON 文件，断网不丢失
历史记录不会丢失，但建议每周导出备份一次。

---

**Q33: 安装后怎么确认成功了？**
A: 按以下 6 步验证，每步都有预期输出对照：
1. 启动 `python run.py` → 看到 `Running on http://127.0.0.1:5001`
2. 访问 `/health` → 返回 `{"success": true, "status": "ok"}`
3. 访问 `/api/system/version` → 返回版本号和 API 数量(102)
4. 查询股票信息 → 返回名称+价格
5. 获取 K 线 → 返回 OHLCV 数组
6. 运行回测 → 返回收益率/夏普比率
详见 SKILL.md「开箱即用验证清单」。任何步骤失败可查「错误码速查索引」一步解决。
