## 垂直领域适配

### Q1：技术团队的报告和销售团队的报告有什么区别？
**答**：报告模板会根据用户档案中的"部门"字段自动适配。
- **技术团队**：侧重开发进度、Bug 数量、技术方案选择、代码审查、上线记录
- **销售团队**：侧重客户拜访量、签约额、回款进度、商机转化率、竞品动态
- **管理层**：侧重目标达成率、团队状态、风险预警、资源需求

如果部门未填写，AI 会从工作内容中推测（"修复了3个BUG"→技术背景，"见了5个客户"→销售背景）。

### Q2：我是产品经理，日报需要同时写技术进度和需求进展，模板能混搭吗？
**答**：AI 会根据你提供的工作内容自动混合对应模块，而非生硬套用单一模板。例如你提到"评审了技术方案+画了3个原型+和运营对了一版需求"，报告会同时输出"研发协作"和"产品设计"两个子章节。

### Q3：可以生成双语报告吗（中英双语）？
**答**：当前 v3.1 版本以中文为主要输出语言。如果需要英文版本，可以直接说："同时生成英文版本"。AI 将基于同一份结构化数据，生成中英双栏对照版本。

## 工具与平台兼容

### Q4：支持用飞书 / 钉钉 / 企业微信发送报告吗？
**答**：v3.1 版本以邮件为主要发送渠道。企业微信发送可以使用 WorkBuddy 的 `wecomcli-msg` 技能配合使用。飞书/钉钉推送计划在 v5.0 版本实现。当前可以先生成文件再手动上传。

### Q5：报告能直接导出到在线文档（腾讯文档、Notion）吗？
**答**：v3.1 版本生成的是本地文件（Word/Excel/PPT/PDF/Markdown）。导出 Markdown 后可以方便地复制到 Notion、语雀等平台。腾讯文档直出计划在后续版本实现。

### Q6：在 macOS 和 Windows 上体验有差异吗？
**答**：核心功能一致。差异体现在：
- Word 生成：macOS 上 python-docx 行为一致
- 文件路径：macOS 用 `/`，Windows 用 `\`，脚本自动处理
- 邮件授权码存储：macOS 可用 Keychain，Windows 用 fernet 加密文件

### Q7：没有安装 Office 软件，能打开生成的 Word/Excel/PPT 吗？
**答**：可以。生成的 docx/xlsx/pptx 文件是 Office Open XML 格式，以下工具均可打开：
- WPS（免费，国产推荐）
- 腾讯文档、石墨文档（在线打开）
- LibreOffice（免费开源）
- Google Docs / Sheets / Slides（网页版）

## 定制化使用

### Q8：能自定义报告模板吗？
**答**：可以。将自定义 Markdown 模板放入 `templates/custom/` 目录，命名为 `{your_name}_daily.md` 等。使用时说"用我的 XX 模板生成周报"，AI 会自动匹配。

### Q9：我的公司有标准报告格式（带 logo、固定页眉页脚），能适配吗？
**答**：可以。在 `config/user_profile.md` 中填写公司名称和 logo 路径。HTML 邮件模板和 Word 报告会自动加入公司标识。复杂版式需求可手动在生成后微调。

### Q10：可以按"周几"或"每月几号"自动生成报告吗？
**答**：目前通过 WorkBuddy 的自动化（automation）机制可实现。设置每天/每周固定时间触发 Skill，自动生成本日/本周报告并发送。说"帮我设置每天18:00自动生成日报并发给张总"即可配置。

## 数据安全

### Q11：我的工作内容会被 AI 服务商看到吗？
**答**：报告内容由你本地运行的 WorkBuddy 处理（如配置了本地 LLM）。如果使用云端模型，建议不要在报告中写入公司机密数据。内部数据可使用本地 LLM（如你配置的 `http://192.168.2.198:4000/v1`）。

### Q12：邮箱密码/授权码是如何存储的？
**答**：v3.1 默认不存储密码明文。`setup_email.py` 使用 `cryptography.fernet` 加密存储（需 `pip install cryptography`）。KEY 文件单独存放于 `config/.fernet_key`，建议不纳入版本控制。

### Q13：发送给多人的报告如何区分保密级别？
**答**：AI 会根据收件人类型自动调整：
- **直属领导**：完整版（含数据、分析、反思）
- **团队成员**：协作版（侧重协作事项和进度同步）
- **全员公告**：公开版（去除敏感数据和内部评估）

如需精细控制，在请求时添加"仅发送本季度数据，隐去客户名称"等指令。

## 使用技巧与最佳实践

### Q14：怎样让报告写得更好？
**答**：垃圾进垃圾出。提供工作内容时遵循"具体事项+量化数据+做了什么"三要素：
- ❌ "开发了功能"
- ✅ "完成了用户登录模块：实现了手机号+验证码登录，前端用 React Hooks，后端 Node.js + Redis 缓存，已通过单元测试覆盖率达 90%"

### Q15：补写昨天的日报怎么办？
**答**：直接说"帮我补写昨天的日报"，提供昨天的工作内容。报告日期会自动标注为昨天。

### Q16：多个项目并行，能按项目分别生成报告吗？
**答**：可以。说"按项目分组生成日报"或"单独生成项目A的进度报告"。AI 会从工作内容中识别项目标签并分组输出。

## 故障排查

### Q17：发送邮件一直失败，怎么办？
**答**：按以下顺序排查：
1. **授权码 vs 密码**：确认使用的是授权码（163/QQ/Gmail）而非邮箱登录密码
2. **SMTP 服务是否开启**：登录邮箱网页版确认 SMTP 服务已启用
3. **端口和加密**：SSL 用 465，STARTTLS 用 587
4. **企业邮箱**：确认管理员已开放 SMTP 权限
5. **Gmail/Outlook 在境内**：可能需要代理，建议暂时切换至国内邮箱
6. 查看 `logs/email_send_log.md` 获取详细错误信息

### Q18：生成的文件打开乱码？
**答**：Word/Excel/PPT 使用 UTF-8 编码。如遇乱码：
- Word：用 WPS 或 Office 2016+ 打开
- Excel/CSV：用 Excel → 数据 → 自文本/CSV → 选择 UTF-8 编码
- 纯文本：用 VS Code 或记事本打开（确认编码为 UTF-8）
