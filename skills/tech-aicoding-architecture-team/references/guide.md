# AICoding 架构专家团

`$tech-aicoding-architecture-team` AICoding 架构专家团总入口

## 来源说明

- WorkBuddy 分类：技术工程
- WorkBuddy 卡片：AICoding 架构专家团
- 作者/来源：WorkBuddy
- 卡片摘要：面向复杂系统架构设计，协同完成资料摄入、调研、业务、系统、部署、安全与用户故事全流程交付。
- 标签：架构设计、文档交付、专家协作
- 提取状态：截图卡片可见团队名、摘要和标签；本地 WorkBuddy marketplace 包未包含完整专家详情 payload。

## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `context-ingestion` | 资料摄入、现状梳理、约束提取和问题归纳 |
| `architecture-research` | 技术调研、竞品/同类方案、架构模式和选型依据 |
| `business-modeling` | 业务域、用户旅程、流程、实体和规则建模 |
| `system-design` | 服务划分、数据模型、接口、事件、权限和扩展性设计 |
| `deployment-security` | 部署拓扑、环境、CI/CD、安全、合规和运维基线 |
| `user-story-delivery` | 用户故事、验收标准、里程碑、风险和交付文档 |
| `adr-package` | ADR、架构图、取舍说明和评审材料 |

## 与已有专家团的边界

| 已有专家团 | 主要关系 |
|---|---|
| `tech-code-refactoring` | 覆盖既有系统重构；本团更偏复杂系统从资料到架构交付的前期设计。 |
| `tech-software-development` | 覆盖开发执行；本团侧重架构文档、用户故事和设计交付。 |
| `tech-api-documentation` | 覆盖 API 文档；本团可定义 API 边界但不会替代完整接口文档专项。 |

## 我可以帮你做这些

1. 资料摄入

   整理 PRD、现状系统、访谈、代码结构、运行约束和未知问题。

2. 调研与建模

   完成技术调研、业务域模型、流程、实体、权限和关键场景。

3. 系统架构

   设计模块/服务、数据、接口、事件、集成、扩展性和可靠性。

4. 部署与安全

   规划环境、部署拓扑、CI/CD、安全边界、合规和运维基线。

5. 交付文档

   输出用户故事、ADR、架构图、风险、验收标准和实施路线。
