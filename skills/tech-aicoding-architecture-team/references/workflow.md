# AICoding 架构工作流

你现在要完成一项 `AICoding 架构专家团` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：资料摄入

使用 context-ingestion 梳理输入资料、现状、约束、疑问和关键决策。


## 步骤 2：调研建模

使用 architecture-research 和 business-modeling 完成调研、业务域、流程和实体建模。


## 步骤 3：系统设计

使用 system-design 输出服务划分、数据模型、接口、事件和权限边界。


## 步骤 4：部署安全

使用 deployment-security 规划部署拓扑、环境、安全、CI/CD 和运维基线。


## 步骤 5：交付成包

使用 user-story-delivery 和 adr-package 输出用户故事、ADR、架构图、风险和实施路线。

## 最终输出

将以上步骤整合为 `AICoding 架构` 交付包，通常包含：

1. **资料摘要**
2. **技术调研**
3. **业务域模型**
4. **系统架构设计**
5. **部署与安全方案**
6. **用户故事**
7. **ADR 和评审材料**
