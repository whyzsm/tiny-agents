# 工程保障工作流

你现在要完成一项 `工程保障团队` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：工程总监定门槛

使用 engineering-director 明确质量目标、发布准入、风险等级和决策人。


## 步骤 2：代码审查

使用 code-reviewer 审查安全、性能、正确性、可维护性和测试缺口。


## 步骤 3：架构 ADR

使用 architecture-adr 审查系统边界、依赖、扩展性、ADR 和替代方案。


## 步骤 4：SRE 准备

使用 sre-readiness 检查可观测性、容量、告警、回滚和应急预案。


## 步骤 5：发布门禁

使用 quality-gate 和 release-risk 输出 go/no-go、阻塞项、风险和责任人。

## 最终输出

将以上步骤整合为 `工程保障` 交付包，通常包含：

1. **工程保障评分**
2. **代码审查表**
3. **架构 ADR 建议**
4. **SRE 准备清单**
5. **发布 go/no-go 建议**
6. **风险与责任人**
