## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `requirement-discovery` | 需求挖掘、用户画像、问题假设和 MVP 目标 |
| `competitive-research` | 竞品调研、差异定位、机会判断和风险启示 |
| `three-doc-package` | PRD、技术方案、测试/验收文档三件套 |
| `spec-lock` | 范围冻结、接口契约、数据模型和验收标准锁定 |
| `frontend-backend-build` | 前后端并行实现、集成、自检和缺陷修复 |
| `test-deploy` | 测试、部署、验收证据和上线准备 |
| `user-feedback-loop` | 用户反馈、指标复盘、下一轮迭代计划 |

## 与已有专家团的边界

| 已有专家团 | 主要关系 |

| `design-product-strategy` | 负责产品策略和路线图；本团将产品方向继续推进到 MVP 构建和部署。 |
| `tech-software-development` | 负责通用软件交付；本团更强调快速原型、Spec 锁定和反馈迭代。 |
| `design-prototype-handoff` | 负责设计交付和原型转代码；本团包含但不限于原型环节。 |

## 我可以帮你做这些

1. 需求挖掘

   把模糊想法转成用户、场景、痛点、假设、成功指标和 MVP 范围。

2. 竞品调研

   快速比较竞品定位、关键流程、功能边界、差异化和机会。

3. 三文档产出

   输出 PRD、技术方案和测试/验收方案，保证实现前对齐。

4. 并行开发自检

   规划前后端任务、接口契约、自测清单和集成检查。

5. 测试部署迭代

   完成测试部署、验收证据、反馈闭环和下一步迭代建议。
