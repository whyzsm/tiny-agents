# MVP 开发工作流

你现在要完成一项 `MVP 开发专家团` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：需求挖掘

使用 requirement-discovery 明确用户、问题、目标、假设、MVP 边界和成功指标。


## 步骤 2：竞品调研

使用 competitive-research 生成竞品矩阵、差异化、风险和 MVP 取舍。


## 步骤 3：三文档

使用 three-doc-package 输出 PRD、技术方案、测试/验收方案。


## 步骤 4：Spec 锁定

使用 spec-lock 固定接口、数据、交互、验收和非目标。


## 步骤 5：开发测试部署

使用 frontend-backend-build、test-deploy 和 user-feedback-loop 完成构建、自检、上线和反馈闭环。

## 最终输出

将以上步骤整合为 `MVP 开发` 交付包，通常包含：

1. **MVP 范围**
2. **竞品调研**
3. **PRD/技术方案/测试方案**
4. **Spec 锁定表**
5. **开发计划**
6. **测试部署清单**
7. **反馈迭代计划**
