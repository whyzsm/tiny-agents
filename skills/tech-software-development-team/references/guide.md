## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `product-requirements` | 需求澄清、用户故事、验收标准和范围冻结 |
| `architecture-design` | 系统架构、模块边界、数据流、接口和技术选型 |
| `task-breakdown` | 按里程碑、模块、风险和依赖拆解工程任务 |
| `parallel-implementation` | 多工程师并行实现、接口对齐和集成节奏 |
| `qa-validation` | 质量验证、测试证据、缺陷回归和验收结果 |
| `delivery-management` | 排期、风险、发布说明、交付状态和后续计划 |

## 与已有专家团的边界

| 已有专家团 | 主要关系 |
|---|---|
| `design-product-strategy` | 负责产品方向、路线图和指标；本团接住明确目标后的工程交付。 |
| `tech-code-refactoring` | 负责已有代码重构；本团负责新需求或小需求的端到端开发交付。 |
| `tech-test-automation` | 负责测试自动化专项；本团只把测试作为开发流程的一环。 |

## 我可以帮你做这些

1. 需求冻结

   整理目标、用户故事、非目标、验收标准、开放问题和最小交付范围。

2. 架构与拆工

   确定架构、接口、数据模型、模块边界、依赖和任务负责人。

3. 并行开发

   按模块生成实现计划、接口契约、集成顺序和冲突规避策略。

4. QA 验证

   建立测试用例、冒烟路径、回归范围和验收证据。

5. 交付管理

   输出发布说明、风险、回滚、状态同步和后续 backlog。
