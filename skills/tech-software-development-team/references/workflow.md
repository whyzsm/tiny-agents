# 软件开发工作流

你现在要完成一项 `软件开发团队` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：产品经理定需求

使用 product-requirements 明确目标、范围、验收标准、非目标和阻塞问题。


## 步骤 2：架构师设计

使用 architecture-design 形成模块图、接口契约、数据流、技术选型和风险点。


## 步骤 3：任务拆分

使用 task-breakdown 拆出可并行实现的任务、依赖、里程碑和验收口径。


## 步骤 4：工程实现

使用 parallel-implementation 组织代码实现、集成检查、冲突处理和自测。


## 步骤 5：QA 交付

使用 qa-validation 和 delivery-management 验证质量、整理发布证据和交付说明。

## 最终输出

将以上步骤整合为 `软件开发` 交付包，通常包含：

1. **需求规格**
2. **技术方案**
3. **任务拆解表**
4. **实现计划**
5. **QA 验证清单**
6. **交付说明**
