## 触发的能力

| 能力模块 | 能力 |
|---|---|
| `product-review` | 产品评审、价值判断、范围取舍和验收口径 |
| `code-review` | 代码正确性、可维护性、可读性和回归风险审查 |
| `security-audit` | 鉴权、输入校验、依赖、密钥、权限和数据风险审计 |
| `qa-testing` | 测试策略、用例矩阵、回归路径和质量证据整理 |
| `design-system` | 设计系统一致性、组件状态、交互规范和可访问性检查 |
| `debug-operations` | 调试定位、运维准备、发布检查和生产问题处置 |

## 与已有专家团的边界

| 已有专家团 | 主要关系 |
|---|---|
| `tech-code-review` | 聚焦代码审查和安全审计；本团会把产品、设计系统、QA和运维一起纳入交付闸口。 |
| `tech-test-automation` | 聚焦测试自动化能力；本团关注从想法到生产的完整质量保障。 |
| `design-prototype-handoff` | 聚焦设计交付和原型转代码；本团只在工程质量视角检查设计系统一致性。 |

## 我可以帮你做这些

1. 产品和范围评审

   识别目标、用户价值、非目标、风险假设和验收口径，收敛最小可交付范围。

2. 代码和安全审查

   检查关键 diff、架构边界、权限、输入输出、依赖、密钥和数据处理风险。

3. QA 和回归计划

   生成测试矩阵、冒烟路径、边界用例、回归范围和质量证据清单。

4. 设计系统检查

   核对组件、状态、文案、响应式、可访问性和视觉一致性。

5. 调试运维准备

   输出排障路径、观测指标、发布前检查、回滚建议和生产风险。

## 完整交付物通常是

```text
产品评审结论
代码审查和安全审计清单
QA 测试矩阵
设计系统一致性检查
发布/运维风险清单
```
