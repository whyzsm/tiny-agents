# 软件工坊工作流

你现在要完成一项 `软件工坊专家团` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：产品评审

使用 product-review 明确业务目标、P0/P1/P2 范围、验收标准和风险假设。


## 步骤 2：设计系统和架构检查

使用 design-system 和 code-review 检查组件一致性、模块边界、依赖和复杂度。


## 步骤 3：代码审查和安全审计

使用 code-review 和 security-audit 审查正确性、权限、输入输出、敏感信息和依赖风险。


## 步骤 4：QA 验证

使用 qa-testing 输出测试矩阵、冒烟路径、边界用例、自动化建议和证据要求。


## 步骤 5：调试运维

使用 debug-operations 准备日志、指标、告警、发布检查、回滚和问题定位路径。

## 最终输出

将以上步骤整合为 `软件工坊` 交付包，通常包含：

1. **产品评审结论**
2. **代码审查和安全审计清单**
3. **QA 测试矩阵**
4. **设计系统一致性检查**
5. **发布/运维风险清单**
