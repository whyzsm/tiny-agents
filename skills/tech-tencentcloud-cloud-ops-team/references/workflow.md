# 腾讯云技术支持工作流

你现在要完成一项 `腾讯云技术支持专家团` 任务。本包从 WorkBuddy 技术工程卡片转换而来，请按任务范围选择必要步骤，不要机械输出无关文档。

## 步骤 1：CloudQ 治理

使用 cloudq-governance 汇总多云资源、账号、成本、安全和架构风险。


## 步骤 2：AndonQ 工单

使用 andonq-ticketing 分类问题、确定优先级、提取证据和生成问答回复。


## 步骤 3：MigraQ 迁移支持

使用 migration-support 分析迁移范围、路径、依赖、风险和验证方案。


## 步骤 4：架构可视化

使用 architecture-visualization 输出云资源拓扑、关键链路和故障域。


## 步骤 5：运维验证

使用 cloud-ops-qa 完成变更建议、恢复验证、FAQ 和 runbook。

## 最终输出

将以上步骤整合为 `腾讯云技术支持` 交付包，通常包含：

1. **多云治理清单**
2. **工单回复**
3. **排查路径**
4. **迁移支持建议**
5. **架构可视化说明**
6. **运维 FAQ/runbook**
