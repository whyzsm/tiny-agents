# 测试自动化专家团

`$tech-test-automation-team` 测试自动化工作流入口

## 触发的技能

| 技能 | 能力 |
|---|---|
| `$tech-test-automation-team` | 完整测试自动化工作流总入口 |
| `$superpowers-tdd` | TDD 策略、失败测试、最小实现、重构闭环 |
| `$test-case-generator` | 从源码生成单元测试、mock、断言、边界与错误路径 |
| `$test-patterns` | Node.js、Python、Go、Rust、Bash 单元/集成测试与覆盖率 |
| `$e2e-testing-patterns` | Playwright/Cypress E2E、关键路径、稳定选择器、CI 与可访问性检查 |
| `$api-test-automation` | REST/GraphQL、契约测试、mock 服务、性能测试与 API 测试报告 |

## 我可以帮你做这些

1. TDD 功能开发

   调用 `superpowers-tdd`

   为功能或缺陷修复建立先失败再实现的测试闭环，并在验证通过后重构。

2. 单元测试生成

   调用 `test-case-generator`

   根据已有源码生成测试文件，覆盖正常路径、边界条件、错误路径、mock 和断言。

3. 单元/集成测试落地

   调用 `test-patterns`

   按项目真实框架编写、运行和调试测试，并补充覆盖率、依赖 mock 和夹具模式。

4. 端到端测试建设

   调用 `e2e-testing-patterns`

   设计浏览器关键用户路径、稳定选择器、CI 执行、降低 flaky 和视觉/可访问性检查。

5. API 自动化测试

   调用 `api-test-automation`

   构建 REST/GraphQL 接口测试、契约测试、mock 服务、性能基线和测试报告。

6. QA 发布质量计划

   调用 `tech-test-automation-team` 的 QA 计划阶段

   输出覆盖矩阵、缺陷严重级别、自动化 ROI、发布准入清单和质量指标看板。

## 完整交付物通常是

```text
测试策略

单元、集成、E2E 和 API 测试代码

测试执行报告

覆盖率和性能基线

QA 覆盖矩阵

发布准入清单和质量看板
```

## 你可以这样用我

```text
$tech-test-automation-team 为这个功能按 TDD 补齐测试并运行验证

$tech-test-automation-team 根据这段源码生成单元测试，覆盖边界和错误路径

$tech-test-automation-team 为登录到下单流程设计 Playwright E2E 测试方案

$tech-test-automation-team 为这些 REST 接口生成契约测试、mock 服务和测试报告
```
