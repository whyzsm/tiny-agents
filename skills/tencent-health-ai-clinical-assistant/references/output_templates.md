# 输出模板（Output Templates）

## 模板 A：疑似诊断报告（无当前诊断 + 最高可能性为"高"）

> **前置条件**：至少 1 条疑似诊断的可能性为"高"。若不满足，禁止使用此模板，强制使用模板 D。

```markdown
## 🔍 疑似诊断分析

### 临床信息摘要
- **主诉**：[提取的主诉]
- **关键阳性发现**：[列出核心阳性症状/体征/检查]
- **关键阴性发现**：[有鉴别意义的阴性结果]

---

### 疑似诊断列表（仅列出高可能性诊断）

#### 1. [诊断名称]（可能性：高）

| 项目 | 内容 |
|------|------|
| **诊断依据** | • [支持依据1]<br>• [支持依据2]<br>• [支持依据3] |
| **不完全符合之处** | • [不支持的发现（如有）] |
| **建议进一步检查** | • [确诊所需的关键检查] |

#### 2. [诊断名称]（可能性：高）
...

---

### ⚠️ 需排除的危急情况
- [危急疾病名称]：[需排除的理由]

---

> **声明**：以上分析仅供临床决策参考，不替代执业医师诊断。
```

## 模板 D：缺失信息报告（信息不足 / 置信度不足）

> **触发条件**（满足任一）：
> 1. 原始输入明显不完整
> 2. 所有疑似诊断的最高可能性仅为"中"或"低"（R08 置信度门控触发）

```markdown
## 📋 诊断信息缺失分析

### 当前可用信息摘要
- [已有信息概要]

### 诊断困难原因
[简述为何无法明确诊断。若因置信度不足触发，说明：
"当前信息仅支持中/低置信度的诊断假设，准确度不足以作为诊断输出，需补充关键信息后重新评估。"]

---

### 缺失信息清单

| # | 缺失类型 | 缺失内容 | 理由 | 依据路径 |
|---|---------|---------|------|---------|
| 1 | 实验室指标 | [具体缺什么] | [为什么需要] | `症状A + 体征B → 疑似C → 确诊需D` |
| 2 | 影像学病灶 | [具体缺什么] | [为什么需要] | `指标E异常 → 定位需F检查` |
| 3 | 症状 | [具体缺什么] | [为什么需要] | `现有表现不足以区分G和H` |

**缺失类型说明**：
- **症状**：病史中未采集或未明确的症状信息
- **实验室指标**：未完成或未提供的化验检查
- **体征**：未执行或未记录的体格检查发现
- **影像学病灶**：未完成的影像学检查或未明确描述的病灶
- **解剖部位**：病变定位不明确

---

### 建议下一步
1. [优先级最高的补充检查/采集]
2. [次优先级]
3. [其他建议]

---

### 初步方向提示（仅当最高可能性为"中"时出现）

> ⚠️ **以下仅为初步方向提示，非诊断结论，需补充上述缺失信息后重新评估。**
>
> 当前信息初步提示 [方向描述]，但置信度不足以作为诊断输出。

[若最高可能性为"低"，删除此段，不提供方向提示]

---

> **声明**：以上分析仅供临床决策参考，不替代执业医师诊断。
```

## 输出路由逻辑

```
输入信息
  │
  ├── 有当前诊断？
  │     ├── 是 → 输出模板B（鉴别诊断）+ 模板C（误诊/漏诊风险）
  │     │
  │     └── 否 → Phase 2 推理后，评估最高疑似诊断的可能性
  │              │
  │              ├── 最高 = 高 → 输出模板A（疑似诊断，仅高可能性条目）
  │              │
  │              ├── 最高 = 中 → 输出模板D（缺失信息 + 方向提示）
  │              │               ⛔ 禁止输出模板A
  │              │
  │              └── 最高 = 低 → 输出模板D（仅缺失信息，不附方向提示）
  │                              ⛔ 禁止输出模板A
  │
  └── 任何情况下，若发现红旗征象 → 在输出开头插入 ⚠️ 急症提示
```
