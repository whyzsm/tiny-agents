# Style Presets

这些预设用于快速确定设计方向。不要机械套用；先看产品类型、目标用户和当前技术栈，再选择一个主方向。

## 1. Operational Dense

适合：T-MAX 后台、运营管理、客服工单、网点/分拨高频操作页。

- 关键词：克制、高密度、可扫描、低干扰
- 布局：紧凑搜索区 + 高信息密度表格 + 清晰状态色
- 色彩：中性色为主，主色只强调关键操作
- 字体：清晰、稳定、数字易读
- 避免：大 hero、大装饰图、过度圆角、过度阴影

## 2. Data Command

适合：看板、调度大屏、指标分析、异常监控。

- 关键词：实时、层级、趋势、预警
- 布局：指标条 + 分区图表 + 异常列表
- 色彩：深浅均可，状态色必须可读
- 记忆点：核心 KPI 或异常信号要第一眼可见
- 避免：只做炫酷背景，不突出数据关系

## 3. Trustworthy SaaS

适合：管理控制台、配置中心、权限平台、企业工具。

- 关键词：专业、稳定、可信、清晰
- 布局：稳定导航 + 表单分组 + 明确保存反馈
- 色彩：低饱和主色 + 清晰状态色
- 交互：强 loading / disabled / error 反馈
- 避免：营销化文案和过多动效

## 4. Editorial Landing

适合：对外介绍页、活动页、方案页、产品说明。

- 关键词：叙事、节奏、品牌感、记忆点
- 布局：首屏重点信息 + 分段叙事 + 证据区
- 色彩：允许更强视觉方向，但要控制对比度
- 记忆点：首屏必须有一个可被记住的视觉锚点
- 避免：通用紫蓝渐变、无业务含义插画、空泛大标题

## 5. Focused Tool

适合：单一任务工具、编辑器、上传/转换/生成类小应用。

- 关键词：专注、直接、即时反馈
- 布局：输入区 + 操作区 + 结果区，减少页面装饰
- 色彩：让主要操作和结果状态更突出
- 交互：输入校验、进度、失败恢复必须完整
- 避免：把简单工具做成复杂 dashboard

## 选择规则

- 后台高频操作优先 `Operational Dense`
- 数据趋势和监控优先 `Data Command`
- 配置和管理平台优先 `Trustworthy SaaS`
- 对外表达优先 `Editorial Landing`
- 单一工具优先 `Focused Tool`
