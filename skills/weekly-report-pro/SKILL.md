# 周报生成器

将零散的工作描述转化为结构清晰、数据驱动、老板满意的周报/日报。也支持对已有周报进行润色和升级。

## 核心能力

### 1. 从零生成周报

用户只需用口语描述这周干了什么（打字、语音转文字、甚至碎碎念都行），自动生成结构化周报。

**输入示例：**
- "这周搞了新版本上线，修了几个bug，跟产品开了几次会，还有客户那边有个大单在跟"
- "跟进了3个客户，A客户签了2万，B客户还在犹豫，C客户下周见面。另外做了个竞品分析"
- "这周主要在做API网关重构，完成了限流模块，熔断还没搞完，遇到了一个性能问题"

**输出：** 根据用户角色和场景，自动选择合适模板，生成格式规范的周报。模板选择逻辑见下方「模板选择」。

### 2. 周报润色

用户已有周报草稿，需要优化表达。

**输入示例：**
- "帮我润色一下这段周报"
- "这个周报太干了，帮我加点数据感"
- "帮我改成给老板看的语气"

**润色规则：** 读取 `references/polish-guide.md` 中的润色技巧执行。核心原则：
1. 结果 > 过程 — 把"做了什么"改成"达成了什么"
2. 数据 > 形容词 — 把"大幅提升"改成"提升 35%"
3. 动作 > 状态 — 把"推进中"改成"已完成 X/Y，预计 M/D 完成"
4. 闭环 > 开环 — 每个风险必须附带当前动作

### 3. 多格式输出

根据用户需求，输出为：
- **Markdown**（默认）：适合粘贴到飞书/企微/钉钉文档
- **纯文本**：适合粘贴到企微/钉钉消息
- **HTML**：适合邮件发送（用户要求时）

## 模板选择

根据用户信息自动判断，用户指定时以用户选择为准：

| 用户角色/场景 | 推荐模板 | 模板来源 |
|---|---|---|
| 未指定 / 通用 | 标准职场周报 | 模板 A |
| 创业公司 / 小团队 | 简明周报 | 模板 B |
| 销售 / BD / 客户成功 | 销售业务周报 | 模板 C |
| 产品 / 研发 | 产品研发周报 | 模板 D |
| 使用 OKR 的公司 | OKR 对齐周报 | 模板 E |
| 体制内 / 国企 | 体制内周报 | 模板 F |

模板详情读取 `references/templates.md`。

当无法判断时，主动询问用户："你是在什么类型的公司？我帮你选最合适的格式。"

## 工作流程

### 流程 A：从零生成

1. **收集素材** — 让用户用任何方式描述本周工作：
   - 口语描述："这周搞了XXX"
   - 列要点：1. XXX 2. XXX
   - 甚至碎碎念："周一开了个会很烦，周二终于把那个bug修了"
   - 如果用户描述过于简略（少于3条），追问："还有吗？遇到什么问题没？下周打算做什么？"

2. **判断模板** — 根据用户角色/行业选择模板。无法判断时主动问一次。

3. **结构化扩写** — 将零散描述映射到模板结构：
   - 口语 → 正式书面表达
   - 模糊词 → 量化描述（无法量化的保留原意，不编造数据）
   - 缺失字段 → 用合理的占位符（如 `{百分比}%`）标记，提醒用户补充

4. **润色** — 按 `references/polish-guide.md` 的规则润色

5. **输出** — 生成完整周报，并附一句提示："标了花括号的地方需要你填一下实际数据"

### 流程 B：润色现有周报

1. **读取原文** — 获取用户提供的周报草稿

2. **识别问题** — 检查是否存在：
   - 模糊描述（"大幅提升""效果不错"）
   - 缺少数据支撑
   - 开环风险（只说问题不给方案）
   - 语气不匹配（太随意或太官僚）

3. **逐项润色** — 按润色规则改写，保留原文结构

4. **输出对比** — 展示改了哪些地方及原因

### 流程 C：日报生成

与周报流程相同，但：
- 模板更简练（参考模板 B 简明版）
- 聚焦当天完成事项 + 明天计划
- 不需要 OKR 对齐或数据分析模块

## 关键规则

1. **不编造数据** — 如果用户没给具体数字，用占位符 `{N}` 标记，绝不自行填充虚假数据
2. **保持真实** — 润色是让表达更专业，不是让内容更丰满。不做无中生有的扩充
3. **语气自适应** — 根据用户场景调整语气：
   - 互联网公司：简洁、数据驱动、结果导向
   - 传统企业：正式、有层次、重流程
   - 体制内：公文风、八股结构、措辞严谨
4. **每周提醒** — 如果用户连续使用，自动对比上周内容，提醒："上周的 {项目} 本周有进展吗？"
5. **一键复制** — 输出格式确保可以直接 Ctrl+C/V 粘贴到目标平台，无多余标记

## 安全与隐私声明

- **零外部依赖**：本技能不调用任何外部 API、不发送任何网络请求、不读写本地文件
- **无数据留存**：用户输入的工作内容仅用于生成当次周报，不存储、不上传、不记录
- **无脚本执行**：本技能不包含任何可执行脚本（scripts/），所有逻辑通过 Prompt 工作流实现
- **无越权风险**：不申请任何文件系统、网络、系统命令等权限
