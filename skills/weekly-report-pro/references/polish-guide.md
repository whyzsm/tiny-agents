# 周报润色技巧

## 核心原则

### 1. 结果 > 过程
- 差：本周组织了3场会议，讨论了产品方案
- 好：本周完成产品方案定稿，产出PRD v2.1，3个模块达成一致

### 2. 数据 > 形容词
- 差：用户增长很快
- 好：用户增长 23%，新增 1,247 人（上周 1,014 人）

### 3. 动作 > 状态
- 差：项目正在进行中
- 好：项目进入测试阶段，已完成 87% 用例执行，发现 3 个 P1 缺陷

### 4. 闭环 > 开环
- 差：发现系统响应慢
- 好：发现系统响应慢（P99 从 200ms 升至 1.2s），已定位到 DB 慢查询，下周优化索引

## 常见润色操作

### 量化模糊描述

| 原文 | 润色后 |
|------|--------|
| 大幅提升 | 提升 35% |
| 基本完成 | 完成率 95%，剩余 2 项待确认 |
| 效果不错 | 转化率从 2.1% 提升至 3.8% |
| 推进中 | 已完成 X/Y 项，预计 M/D 完成 |
| 多次沟通 | 与 X 部门沟通 3 次，达成 Y 共识 |

### 添加业务价值

每项工作后面追问："所以呢？"

- 修复了 5 个 bug → 修复 5 个线上 bug，客诉率下降 18%
- 优化了部署流程 → 部署耗时从 40min 降至 12min，发布效率提升 3.3 倍
- 完成了培训 → 完成 30 人新产品培训，预计下周上线后减少 50% 咨询量

### 风险描述公式

风险 = 现象 + 影响范围 + 当前动作 + 需要的支持

- 差：第三方接口不稳定
- 好：第三方支付接口本周超时 3 次（影响约 200 笔订单），已联系对方技术排查，需 CTO 出面推进 SLA 升级

## 语气调整

### 向上汇报（给老板看）
- 砍掉过程细节，保留结果和风险
- 数字前置，方法后置
- 每段不超过 3 行

### 平级同步（给同事看）
- 保留协作细节和接口人
- 标注阻塞项和依赖
- 写清楚时间节点

### 向下通报（给团队看）
- 多写方向和背景
- 明确优先级排序
- 写清楚下周目标而非动作
